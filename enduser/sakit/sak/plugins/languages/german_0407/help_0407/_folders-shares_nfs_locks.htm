Vorheriges Thema

NFS-Sperren

NFS-Sperren ermöglichen einem Prozess exklusiven Zugriff auf einen Teil einer Datei oder die gesamte Datei. Die Dateisperrung wird sowohl auf der Serverappliance als auch auf dem Client implementiert. Bei einer gesperrten Datei wird der Puffercache für diese Datei nicht verwendet, und jede Schreibanforderung wird unmittelbar an den Server gesendet.

Wenn die Serverappliance nach einem Systemausfall neu gestartet wird, versucht sie, den vorherigen Sperrstatus der Datei wiederherzustellen. Fällt der Client aus, hebt die Serverappliance die Dateisperre auf. Nach dem Neustart des Clients steht ein kurzer Zeitraum zur Wiederherstellung der Dateisperre zur Verfügung.

So verwalten Sie NFS-Sperren

  1. Wählen Sie auf der primären Navigationsleiste Freigaben aus.
  2. Wählen Sie Protokolle freigeben aus.
  3. Wählen Sie NFS-Protokoll und dann Eigenschaften aus.
  4. Wählen Sie Sperren aus.
  5. Optional: Wählen Sie in der Liste Aktuelle Sperrungen den Client aus, dessen Sperren aufzuheben sind.
  6. Optional: Geben Sie im Feld Wartezeit in Sekunden die Anzahl an Sekunden ein, die nach einem Neustart vergehen, bis die Serverappliance eine Dateisperre mit einem Client erneut einrichtet.
  7. Klicken Sie auf OK.
