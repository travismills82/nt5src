Vorheriges Thema

NFS

Mittels des NFS-Protokolls kann eine Serverappliance als NFS-Server (Network File System) eingesetzt werden. Benutzer können Dateien in einer gemischten Computer-, Betriebssystem- und Netzwerkumgebung gemeinsam nutzen.

Benutzer von Computern, auf denen die NFS-Clientsoftware ausgeführt wird, können auf die Freigaben der Serverappliance zugreifen, indem Sie diese Freigaben mit ihrem eigenen Computer verbinden oder mounten.

UNIX-Computer verwenden das freiwillige Sperrverfahren (Advisory Locking) für sämtliche Sperranforderungen. Das Betriebssystem erzwingt demnach keine Sperrsemantik für eine Datei, und Anwendungen, welche die Existenz von Sperrungen überprüfen, können diese effizient verwenden. Das NFS-Protokoll implementiert sogar obligatorische Sperren für die über NFS eingegangenen Sperranforderungen. Auf diese Weise ist gewährleistet, dass über NFS eingegangene Sperren über das SMB-Protokoll (Server Message Block) und für Anwendungen, die lokal auf die Dateien zugreifen, sichtbar sind. Das Betriebssystem erzwingt obligatorische Sperren.
