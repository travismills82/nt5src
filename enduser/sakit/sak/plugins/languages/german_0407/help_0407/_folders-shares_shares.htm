Vorheriges Thema

Freigaben

Ein Ordner auf der Serverappliance kann zur gemeinsamen Verwendung mit anderen Computern im Netzwerk freigegeben werden, wenn auf diesen ein Microsoft Windows- oder UNIX-Betriebssystem ausgeführt wird.

Obgleich sich die verfügbaren für Dateifreigabeprotokolle je nach Hersteller der Serverappliance unter Umständen unterscheiden, kann eine Serverappliance folgende Protokolle unterstützen:

  • Windows (CIFS) - Das CIFS-Protokoll (Common Internet File System) wird von Clients verwendet, auf denen ein Windows-Betriebssystem ausgeführt wird.
  • NFS - Das NFS-Protokoll (Network File System) wird von UNIX-Clients verwendet.
  • FTP - Das FTP-Protokoll (File Transfer Protocol) stellt eine alternative Möglichkeit für den Zugriff auf eine Dateifreigabe über ein beliebiges Betriebssystem dar.
  • HTTP - Das HTTP-Protokoll (Hypertext Transfer Protocol) wird verwendet, um mit Webbrowsern auf eine Dateifreigabe zuzugreifen.
  • NetWare - Hierbei handelt es sich um das Protokoll, mit dem NetWare-Clients auf eine Dateifreigabe zugreifen.
  • AppleTalk - Hierbei handelt es sich um das Protokoll, mit dem Macintosh-Clients auf eine Dateifreigabe zugreifen.

Wenn Sie eine Freigabe auf der Serverappliance erstellen, können Sie eins oder alle der aktuell auf der Serverappliance aktivierten oder installierten Protokolle aktivieren.
