Webprotokoll (HTTP)

Bei HTTP (Hypertext Transfer Protocol) handelt es sich um ein Kommunikationsprotokoll, das für die Übertragung von Hypertextdokumenten zwischen Computern über das Web konzipiert ist. Durch HTTP werden die Aktionen definiert, die Webserver und -browser als Reaktion auf verschiedene Befehle durchführen müssen.

WWW-Server

Die im Web verwendeten Befehle werden durch HTTP festgelegt.

Die Ressourcenstandorte werden von HTTP unter Verwendung von URLs (Uniform Resource Locators) angegeben. URLs entsprechen einer Namenskonvention zur eindeutigen Kennzeichnung des Standortes eines Computers, Verzeichnisses bzw. einer Datei im Internet. Der URL gibt ebenfalls das zum Abrufen der Ressource benötigte Internetprotokoll an, z. B. FTP oder HTTP. Wenn der URL einer Ressource bekannt ist, können Sie diesen angeben oder eine Verknüpfung zwischen dem URL und einem Dokument einrichten, das Webbenutzern zur Verfügung gestellt werden soll.

Das HTTP-Protokoll unterstützt anonymen Zugriff sowie Standardauthentifizierung und Windows-Authentifizierung.

So konfigurieren Sie die Eigenschaften von Webfreigaben (HTTP)

  1. Wählen Sie auf der primären Navigationsleiste Freigaben aus.
  2. Wählen Sie Protokolle freigeben aus.
  3. Wählen Sie HTTP-Protokoll und dann Eigenschaften aus.
  4. Ermöglichen Sie allen IP-Adressen den Zugriff auf Datenfreigaben der Serverappliance, oder wählen Sie in der Liste eine einzelne IP-Adresse aus.
  5. Geben Sie die Portnummer ein, die für den Zugriff auf Datenfreigaben der Appliance verwendet werden kann. Die standardmäßige Portnummer lautet 80.
