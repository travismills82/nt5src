Vorheriges Thema

DNS-Namensauflösung

Damit Benutzer in einem Intranet oder im Internet auf Ihre Website zugreifen können, müssen Sie über eine eindeutige IP-Adresse verfügen, die den Computer im Netzwerk identifiziert. Diese Adresse besteht aus einer langen Folge von Zahlen, die durch Punkte voneinander getrennt sind (beispielsweise 172.16.255.255). Da Benutzer sich numerische Adressen nur schwer einprägen können, werden ihnen mit Namen oder angezeigten Namen wie www.microsoft.com Adressen bereitgestellt, die sie sich problemlos merken können. Die Namensauflösung umfasst die Auflösung des im Clientbrowser eingegebenen angezeigten Namens in die entsprechende numerische Adresse.

Systeme zur Namensauflösung

Windows-Netzwerkomponenten basieren auf der NetBIOS-Namenskonvention. TCP/IP-Komponenten hingegen verwenden eine Namenskonvention, die als Domain Name System (DNS) bezeichnet wird. Unter Windows stimmt der DNS-Hostname der Serverappliance standardmäßig mit dem NetBIOS-Computernamen überein. Die Zuordnung von Computernamen zu IP-Adressen erfolgt nach einer der folgenden Methoden:

  • Statisch - Der Systemadministrator erstellt eine Textdatei für DNS-Namen, die so genannte HOSTS-Datei, oder eine LMHOSTS-Datei für NetBIOS-Namen und gibt den Namen und die IP-Adresse jedes Computers ein. Anschließend wird die Datei über das Netzwerk verteilt. Geht eine Anforderung für eine Verbindung zu einem anderen Computer ein, erfolgt die Auflösung des Namens in die richtige IP-Adresse anhand der Datei. Dieses System ist optimal für einfache Netzwerke, die selten geändert werden.
  • Dynamisch - Wenn ein Clientcomputer eine Verbindung zu einem Netzwerk mit einem DHCP-Server herstellt, weist der DHCP-Server eine Adresse zu und sendet die IP-Adressenzuordnung an einen WINS-Server (Windows Internet Name Service). Der WINS-Server erfasst den Computernamen. Geht eine Anforderung für einen NetBIOS-Computernamen ein, löst der WINS-Server den Namen in die richtige IP-Adresse auf. Die automatische Erkennung und Zuordnung von Computernamen und -adressen reduziert den Verwaltungsaufwand in großen Netzwerken, die häufigen Änderungen unterliegen.

DNS-Namen werden in der Regel unter Verwendung von statischen Informationen aufgelöst. Auf dem DNS-Server befindet sich ein Teil der statischen Datenbank mit Hostnamen und Adressen. Wenn der angeforderte Name nicht in diesem Teil der Datenbank gespeichert ist, sendet der DNS-Server eine Abfrage an einen anderen DNS-Server, um die angeforderte Information abzurufen. Der DNS-Server unter Windows kann so konfiguriert werden, dass er für die Namensauflösung auf niedrigen Ebenen der hierarchischen DNS-Namensstruktur eine Abfrage an einen WINS-Server sendet. Da der WINS-Dienst die Computernamen dynamisch zuweist, wird das statische DNS-System auf diese Weise im Grunde zu einem dynamischen System.
