Vorheriges Thema

Terminaldienste

Unter Verwendung der Terminaldienste können Sie sich an der Serverappliance anmelden und sie über das Netzwerk verwalten, wobei die Verwaltung über einen beliebigen Client möglich ist. Da die Terminaldienste für die Remoteverwaltung installiert werden, sind lediglich zwei gleichzeitige Terminaldienstesitzungen möglich. Durch das Fortbestehen einer Sitzung wird eine Lizenz beansprucht; dies kann sich auf die Arbeit anderer Benutzer auswirken. Wenn zwei Sitzungen ausgeführt werden, wird neuen Benutzern der Zugriff verweigert.

Anmerkung Terminaldienste Advanced Client steht auf der Seite Wartung zur Verfügung und wird nur von Systemen unterstützt, auf denen Internet Explorer ausgeführt wird.

Abmelden und Trennen der Sitzung

Beim Herunterfahren eines Client können Sie über die Terminaldienste entweder die Sitzung trennen oder sich vom System abmelden.

Wenn Sie die Sitzung trennen, wird sie weiterhin auf dem Server ausgeführt. Sie können die Verbindung zum Server wiederherstellen und die Sitzung fortsetzen. Führen Sie einen Vorgang auf einem Server durch, z. B. eine zeitaufwendige Datenbankabfrage, können Sie nach dem Starten des Vorgangs die Sitzung trennen. Zu einem späteren Zeitpunkt können Sie sich erneut am Server anmelden, die Sitzung aufrufen und den Vorgang wieder aufnehmen oder die Ergebnisse überprüfen. Dies ist sinnvoll, wenn Sie einen Vorgang mittels einer RAS-Verbindung über eine gebührenpflichtige Telefonleitung durchführen.

Netzwerkadministratoren sind in der Lage, den Zeitraum zu begrenzen, während dessen eine getrennte Sitzung auf einem Server beibehalten werden kann. Informationen zum Zeitlimit für die Sitzungsdauer erhalten Sie von Ihrem Administrator.

Das Beenden einer Sitzung wird als Abmelden bezeichnet. Durch das Abmelden wird die auf dem Server ausgeführte Sitzung beendet. Sämtliche in der Sitzung ausgeführten Anwendungen werden ebenfalls beendet, und nicht gespeicherte Änderungen an geöffneten Dateien gehen verloren. Bei der nächsten Anmeldung am Server wird eine neue Sitzung erstellt.

Terminaldienste Advanced Client erfordert die Authentifizierung sämtlicher Benutzer, die eine Verbindung herstellen. Aus diesem Grund müssen sich Benutzer bei jedem Sitzungsstart anmelden.

So greifen Sie auf Terminaldienste Advanced Client zu

  1. Wählen Sie auf der primären Navigationsleiste Wartung aus.
  2. Wählen Sie Terminaldienste aus.
  3. Melden Sie sich an.
