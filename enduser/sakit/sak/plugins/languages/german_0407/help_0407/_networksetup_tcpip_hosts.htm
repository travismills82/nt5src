TCP/IP-Hosts

Windows-Netzwerkkomponenten basieren auf der NetBIOS-Namenskonvention. TCP/IP-Komponenten hingegen verwenden eine Namenskonvention, die als Domain Namen System (DNS) bezeichnet wird. Unter Windows stimmt der DNS-Hostname standardmäßig mit dem NetBIOS-Computernamen überein. Die Zuordnung von Computernamen zu IP-Adressen erfolgt nach einer der folgenden Methoden:

  • Statisch - Der Systemadministrator erstellt eine Textdatei für DNS-Namen, die sogenannte HOSTS-Datei, oder eine LMHOSTS-Datei für NetBIOS-Namen, und gibt den Namen und die IP-Adresse jedes Computers ein. Anschließend wird die Datei im Netzwerk verteilt. Geht eine Anforderung für eine Verbindung zu einem anderen Computer ein, erfolgt die Auflösung des Namens in die richtige IP-Adresse anhand der Datei. Dieses System ist optimal für einfache Netzwerke, die selten geändert werden.
  • Dynamisch - Bei Anmeldung eines Clientcomputers weist der DHCP-Server eine Adresse zu und sendet die IP-Adressenzuordnung an einen WINS-Server (Windows Internet Name Service). Der WINS-Server erfasst den Computernamen. Geht eine Anforderung für einen NetBIOS-Computernamen ein, löst der WINS-Server den Namen in die richtige IP-Adresse auf. Die automatische Erkennung und Zuordnung von Computernamen und -adressen reduziert den Verwaltungsaufwand in großen Netzwerken, die häufigen Änderungen unterliegen.

DNS-Namen werden in der Regel unter Verwendung von statischen Informationen aufgelöst. Auf dem DNS-Server befindet sich ein Teil der statischen Datenbank mit Hostnamen und Adressen. Wenn der angeforderte Name nicht in diesem Teil der Datenbank gespeichert ist, ruft der DNS-Server ihn von anderen DNS-Servern ab. Der DNS-Server, auf dem Windows ausgeführt wird, kann so konfiguriert werden, dass er für die Namensauflösung der niedrigen Ebenen der hierarchischen DNS-Namensstruktur eine Abfrage an einen WINS-Server sendet. Da der WINS-Dienst die Computernamen dynamisch zuweist, wird das statische DNS-System auf diese Weise im Grunde zu einem dynamischen System.

Wenn Sie mehrere Web- oder FTP-Sites auf einem einzelnen Server einrichten, die jede über eine eigene IP-Adresse verfügen, treten unter Umständen Probleme mit der automatischen DNS-Registrierung auf. Um die ordnungsgemäße IP-Bindung und DNS-Registrierung sicherzustellen, deaktivieren Sie die automatische DNS-Registrierung von Windows 2000 Server, und konfigurieren Sie die DNS-Registrierung für die Websites manuell. Weitere Informationen zur Deaktivierung und manuellen Konfiguration der automatischen DNS-Registrierung erhalten Sie in der Dokumentation zu Windows 2000 Server.

Wenn Sie einer Serverappliance mehrere Namen zuweisen möchten, müssen Sie für die Serverappliance eine statische Namenszuordnung verwenden. Auf einem Computer können Sie mehrere Namen zu einer IP-Adresse zuordnen oder mehrere Namen verwenden, die jeder über eine eigene IP-Adresse verfügen.

So bearbeiten Sie die HOSTS-Datei

  1. Wählen Sie Globale Einstellungen auf der Seite Netzwerk aus.
  2. Wählen Sie die Registerkarte TCP/IP-Hosts aus. Standardmäßig wird im Feld Host-Dateien die aktuelle Konfiguration der HOSTS-Datei angezeigt.
  3. Bearbeiten Sie die HOSTS-Datei, indem Sie in das Feld klicken und die Daten bearbeiten.
  4. Klicken Sie auf OK.
