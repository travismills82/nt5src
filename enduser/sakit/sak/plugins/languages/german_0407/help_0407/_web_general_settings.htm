Vorheriges Thema

Festlegen allgemeiner Eigenschaften für Siteidentitäten

  1. Geben Sie den eindeutigen Namen der Website im Feld Sitekennung ein.

    Eine Websitekennung muss einheitlich sein und kann bis zu 12 Zeichen in Groß- oder Kleinschreibung enthalten, wobei die folgenden Zeichen ausgenommen sind: " / \ [ ] : ; | = , + * ? < >. Außerdem darf eine Websitekennung nicht ausschließlich aus Punkten (.) oder Leerzeichen bestehen.

  2. Die Serverappliance zeigt den Standardverzeichnispfad automatisch im Feld Verzeichnis an. Sie können diesen Pfad ändern.
  3. Wählen Sie den Benutzerkontentyp aus, den Sie für den Verwaltungszugriff auf die Website verwenden möchten. Sie können ein Domänenkonto oder ein lokales Benutzerkonto erstellen bzw. ein vohandenes Benutzerkonto zuordnen.

    Anmerkung   Sie können ein Domänenkonto nur erstellen, wenn die Serverappliance ein Mitglied der entsprechenden Domäne ist.

  4. Optional: Wenn Sie einem bestehenden Benutzerkonto Administratorrechte zuweisen möchten, geben Sie den Benutzernamen im Feld Websiteadministrator ein.

    Anmerkung   Wenn die Serverappliance Mitglied einer Domäne ist, und Sie einem bestehenden Benutzerkonto Administratorrechte zuweisen möchten, müssen Sie den Domänennamen und den Benutzernamen im Format <Domänenname>\<Benutzername> eingeben.

  5. Optional: Wenn Sie ein Domänenkonto oder ein lokales Benutzerkonto erstellen möchten, geben Sie das Administratorkennwort in das Feld Administratorkennwort ein, und bestätigen Sie es durch die erneute Eingabe in das Feld Kennwort bestätigen. Das Kennwort muss mit den geltenden Kennwortrichtlinien der Domäne übereinstimmen, zu der die Serverappliance gehört.
