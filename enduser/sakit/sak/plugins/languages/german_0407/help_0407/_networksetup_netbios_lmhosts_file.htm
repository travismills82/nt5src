Vorheriges Thema

NetBIOS LMHOSTS-Datei

Namensauflösung

Damit Benutzer auf Ihre Website zugreifen können, müssen Sie über eine eindeutige IP-Adresse verfügen, die den Computer im Netzwerk kennzeichnet. Diese Adresse besteht aus einer langen Folge von Zahlen, die durch Punkte voneinander getrennt sind (z. B. 172.16.255.255). Da numerische Adressen nur schwer zu behalten sind, werden Benutzern mit Namen oder angezeigten Namen wie \\GespeicherteDateien Adressen bereitgestellt, die sie sich problemlos einprägen können. Die Namensauflösung umfasst die Auflösung des im Clientbrowser eingegebenen Anzeigenamens in die entsprechende numerische Adresse. In diesem Abschnitt werden die verschiedenen Systeme zur Namensauflösung beschrieben.

Die Verwendung einer LMHOSTS-Datei ist optional. Wenn keine LMHOSTS-Datei verwendet wird, können jedoch keine Anzeigenamen eingesetzt werden. Stattdessen sind IP-Adressen erforderlich. Dies kann sich als Nachteil erweisen, da Websites im Internet in der Regel DNS verwenden. Wenn Sie einen Domänennamen für Ihre Website registrieren lassen, können Benutzer durch Eingabe dieses Namens in einen Browser auf die Website zugreifen.

Die LMHOSTS-Datei wird gelesen, wenn WINS oder ein Broadcast fehlschlägt und aufgelöste Einträge im Systemcache für den späteren Zugriff gespeichert wurden. Verwendet der Computer den Replikationsdienst ohne WINS, sind Einträge in der LMHOSTS-Datei auf Import- und Exportservern für alle Computer in verschiedenen Subnetzen erforderlich, die an der Replikation teilnehmen.

Die Beispieldatei LMHOSTS.sam, die automatisch im Windows-Verzeichnis installiert wird, kann mit Microsoft Notepad oder einem beliebigen Texteditor bearbeitet werden. Folgende Regeln werden auf Einträge in der LMHOSTS-Datei angewendet:

  • Für jeden Eintrag muss eine gesonderte Zeile verwendet werden.
  • Die IP-Adresse, gefolgt von dem entsprechenden Computernamen, muss in der ersten Spalte beginnen. Bei Einträgen in der LMHOSTS-Datei wird die Groß-/Kleinschreibung nicht unterschieden.
  • Die Adresse und der Computername müssen durch mindestens ein Leer- oder Tabulatorzeichen getrennt sein.
  • Der Beginn eines Kommentars wird in der Regel durch das Nummernzeichen (#) gekennzeichnet. Dieses Zeichen kann jedoch ebenfalls wie im Folgenden beschrieben zur Angabe bestimmter Schlüsselwörter verwendet werden.

Die in der folgenden Tabelle aufgelisteten Schlüsselwörter können in der LMHOSTS-Datei verwendet werden. Beachten Sie jedoch, dass LAN Manager 2.x diese Schlüsselwörter als Kommentare verarbeitet.

Schlüsselwort Definition
#PRE Bewirkt, dass ein Eintrag in den Namencache geladen wird. Dieses Schlüsselwort wird nach einem Eintrag hinzugefügt. Die #PRE-Schlüsselwörter in der LMHOSTS-Datei werden nachgesehen und vor WINS-Lookup zwischengespeichert. #PRE muss Einträgen angefügt werden, die auch in #INCLUDE-Anweisungen enthalten sind; andernfalls wird der Eintrag in #INCLUDE ignoriert.
#DOM:Domäne Verknüpft einen Eintrag mit der angegebenen Domäne (Domäne). Dieses Schlüsselwort wird nach einem Eintrag hinzugefügt und wirkt sich auf das Verhalten des Browsers und der Anmeldedienste in gerouteten TCP/IP-Umgebungen aus. Um einen #DOM-Eintrag zu laden, müssen Sie in der Zeile ebenfalls das #PRE-Schlüsselwort hinzufügen
#INCLUDE Dateiname Zwingt das System, nach dem angegebenen Dateinamen zu suchen und die Datei zu verarbeiten, als ob sie lokal gespeichert wäre. Die Angabe eines Dateinamens, welcher der Universal Naming Convention (UNC) entspricht, ermöglicht die Verwendung einer zentralen LMHOSTS-Datei auf einem Server. Sie müssen den Server vor seinem Eintrag im #INCLUDE-Abschnitt zuordnen und #PRE anfügen, um sicherzustellen, dass er geladen wird. (#INCLUDE wird andernfalls ignoriert).
#BEGIN_ALTERNATE Fasst mehrere #INCLUDE-Anweisungen in einer Gruppe zusammen. Ist eine #INCLUDE-Anweisung erfolgreich, werden alle weiteren #INCLUDE-Anweisungen übersprungen und die Gruppe verlassen.
#END_ALTERNATE Kennzeichnet das Ende einer #INCLUDE-Gruppe.
\0xnn Unterstützt nicht druckbare Zeichen in NetBIOS-Namen. Setzen Sie den NetBIOS-Namen in Anführungszeichen, und geben Sie solche Zeichen als Hexadezimalwert in der \0xnn-Notation an. Auf diese Weise wird sichergestellt, dass benutzerdefinierte Anwendungen, die besondere Namen verwenden, in gerouteten Topologien ordnungsgemäß ausgeführt werden. Das TCP/IP von LAN Manager erkennt kein Hexadezimalformat, so dass bei Verwendung dieser Funktion die Abwärtskompatibilität unter Umständen verloren geht.

Beachten Sie, dass die Hexadezimalschreibweise nur auf ein Zeichen in dem Namen angewendet wird. Dem Namen sollten Leerzeichen hinzugefügt werden, so dass das Sonderzeichen das letzte bzw. 16. Zeichen in der Folge darstellt.

Das folgende Beispiel veranschaulicht die Verwendung dieser Schlüsselwörter.

102.54.94.98    localsrv                 #PRE
102.54.94.97    trey                     #PRE #DOM:netzwerk #PDC
102.54.94.102   "appname        \0x14"   #spezieller Anwendungsserver
102.54.94.123   popular                  #PRE    #source server
#BEGIN_ALTERNATE
#INCLUDE \\localsrv\public\LMHOSTS       #fügt LMHOSTS-Datei von diesem Server hinzu
#INCLUDE \\trey\public\LMHOSTS           #fügt LMHOSTS-Datei von diesem Server hinzu
#END_ALTERNATE

Beachten Sie die folgenden Aspekte dieses Beispiels:

  • Die Server mit den Namen localsrv und trey werden geladen, so dass sie später in einer #INCLUDE-Anweisung in einer zentral verwalteten LMHOSTS-Datei verwendet werden können.
  • Der Servername "appname \0x14" enthält ein Sonderzeichen nach dem 15. Zeichen (einschließlich Leerzeichen), der Name wird demnach in Anführungszeichen gesetzt.
  • Der Server mit dem Namen popular wird basierend auf dem #PRE-Schlüsselwort geladen.

Richtlinien für LMHOSTS-Dateien

Stellen Sie bei Verwendung einer Hosttabellendatei sicher, dass diese stets aktuell und fehlerfrei ist. Gehen Sie gemäß folgenden Richtlinien vor:

  • Aktualisieren Sie die LMHOSTS-Datei, wenn ein Computer ausgetauscht oder aus dem Netzwerk entfernt wird.
  • Verwenden Sie #PRE-Anweisungen, um häufig verwendete Einträge in die Namentabelle des lokalen Computers zu laden. Laden Sie mit Hilfe von #PRE-Anweisungen ebenfalls Server, die in #INCLUDE-Anweisungen enthalten sind.
  • Da LMHOSTS-Dateien von vorne an Zeile für Zeile durchsucht werden, können Sie die Suchgeschwindigkeit für die gängigsten Einträge steigern, indem Sie die Anweisungen der am häufigsten verwendeten Server oben in die Datei schreiben. Dann sollten die Verweise auf die Server folgen, die weniger häufig verwendet werden, und schließlich die Serververweise mit Remote-#INCLUDE-Anweisungen. Geben Sie die #PRE-Einträge am Dateiende ein, da diese Anweisungen beim Systemstart in den Cache geladen werden und später kein Zugriff auf diese Einträge erfolgt. Beachten Sie, dass Kommentarzeilen die Verarbeitungszeit verlängern, da jede Zeile einzeln verarbeitet wird.

So bearbeiten Sie die LMHOSTS-Datei

  1. Wählen Sie auf der primären Navigationsleiste Netzwerk aus.
  2. Wählen Sie Globale Einstellungen aus.
  3. Wählen Sie die Registerkarte NetBIOS-LMHOSTS aus.
  4. Aktivieren Sie das Kontrollkästchen LMHOSTS-Abfrage aktivieren.

    Standardmäßig enthält das Feld unter dem Kontrollkästchen die aktuelle Konfiguration der LMHOSTS-Datei; sind jedoch keine anzuzeigenden Einträge verfügbar, bleibt das Feld leer.

  5. Bearbeiten Sie die LMHOSTS-Datei, indem Sie in das Feld klicken und die Daten ändern.
  6. Klicken Sie auf OK.
