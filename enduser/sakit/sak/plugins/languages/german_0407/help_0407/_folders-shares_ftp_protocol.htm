Vorheriges Thema

FTP-Protokoll

Das FTP-Protokoll (File Transfer Protocol) ist in das Windows-Sicherheitsmodell integriert. Benutzer, die Verbindungen unter Verwendung von FTP herstellen, werden basierend auf den Benutzerkonten der Serverappliance unter Windows authentifiziert. Der Zugriff wird auf Grundlage dieser Benutzerprofile gewährt. Beachten Sie jedoch, dass der FTP-Server Benutzerkennwörter im Netzwerk ohne Datenverschlüsselung überträgt. Folglich könnte ein Benutzer, der über physischen Zugriff auf das Netzwerk verfügt, Benutzerkennwörter während des FTP-Validierungsprozesses untersuchen.

FTP unterstützt alle Befehle der FTP-Clients von Microsoft Windows. Führt eine Serverappliance das FTP-Protokoll aus, können andere Computer mit Hilfe des FTP-Dienstprogramms eine Verbindung zum Server herstellen und Dateien übertragen. FTP-Clientversionen von Drittanbietern hingegen enthalten unter Umständen Befehle, die nicht vom FTP-Serverprotokoll unterstützt werden.
