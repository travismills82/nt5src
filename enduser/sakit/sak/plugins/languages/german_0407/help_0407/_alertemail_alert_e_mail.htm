Einrichten einer Benachrichtigungs-E-Mail

Die Serverappliance kann für die Generierung einer automatischen E-Mail-Benachrichtigung beim Auslösen einer Warnung konfiguriert werden. Sie können festlegen, ob Sie beim Auflösen jeder Warnung benachrichtigt werden möchten, oder lediglich bestimmte Benachrichtigungen, z. B. informierende, warnende oder kritische Benachrichtigungen erhalten wollen.

Bei dieser Funktion werden E-Mails unter Verwendung des Serverappliance-SMTP-Dienstes gesendet. In einer regulären Internetumgebung ist die Konfiguration eines SMTP-Gateways nicht erforderlich. Um E-Mails mit Microsoft® Exchange Server oder Lotus Notes zu senden, müssen Sie jedoch den Namen des spezifischen SMTP-Gateways angeben. Der Servername oder die IP-Adresse des SMTP-Gateways muss im Feld zur Angabe des SMTP-Servers in der Webbenutzeroberfläche eingegeben werden. Den Servernamen des SMTP-Gateways erfahren Sie von Ihrem Microsoft Exchange-Administrator.

So richten Sie die Funktion der Benachrichtigungs-E-Mail ein

  1. Wählen Sie auf der primären Navigationsleiste Wartung aus.
  2. Wählen Sie Benachrichtigungs-E-Mail aus.
  3. Wählen Sie Benachrichtigungs-E-Mail deaktivieren aus.

    - ODER -

    1. Wählen Sie Benachrichtigungs-E-Mail aktivieren aus, und aktivieren Sie anschließend die Kontrollkästchen für die Bedingungen, unter denen die E-Mail zu senden ist.
    2. Geben Sie im Feld An die E-Mail-Adresse des Systemadministrators ein.

      Wenn die Benachrichtigungs-E-Mails an mehrere Adressen zu senden sind, geben Sie die jeweiligen Adressen durch ein Komma getrennt im Feld An ein.

    3. Geben Sie im Feld Mit das SMTP-Gateway oder die IP-Adresse des SMTP-Servers ein.
    4. Klicken Sie zur Überprüfung der Einstellungen auf die Schaltfläche Test.

      Nach dem Klicken auf die Schaltfläche Test wird eine Test-E-Mail gesendet. Wenn der SMTP-Dienst auf Ihrem Computer nicht installiert ist oder kein Netzwerkkabel vorhanden ist, wird die Fehlermeldung "Test E-Mail kann nicht gesendet werden" angezeigt.

      Geht keine Test-E-Mail ein, obwohl auf der Webbenutzeroberfläche die Meldung "Es wurde eine Test-E-Mail gesendet. Sie sollte im Postfach des Administrators ankommen." angezeigt wurde, liegt der Fehler wahrscheinlich beim SMTP-Server. Setzen Sie zur Fehlerbehebung den SMTP-Servernamen zurück.

    5. Klicken Sie auf OK.
