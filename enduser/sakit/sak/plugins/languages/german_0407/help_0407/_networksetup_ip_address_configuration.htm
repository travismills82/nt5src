Vorheriges Thema

IP-Adressenkonfiguration

Jeder Computer im Netzwerk muss über eine eindeutige IP-Adresse verfügen, um Daten senden und empfangen zu können. Auf der Seite IP-Adressenkonfiguration können Sie die Serverappliance für das automatische Abrufen der IP-Adressenkonfiguration vom DHCP-Server (Dynamic Host Configuration Protocol) einrichten. Alternativ besteht die Möglichkeit, IP-Adressen manuell zu konfigurieren.

Darüber hinaus kann die Seite IP-Adressenkonfiguration zur Angabe einer oder mehrerer Gateway-Adressen verwendet werden. Bei einer Gateway-Adresse handelt es sich um die Adresse eines lokalen IP-Routers, der sich in demselben Netzwerk wie die Serverappliance befindet und für die Weiterleitung von Daten an Ziele zuständig ist, die außerhalb des lokalen Netzwerks liegen. Der Wert in jedem Feld muss eine Zahl zwischen 0 und 255 darstellen.

Anmerkung   Bei einer Änderung der IP-Adresse kann der Client unter Umständen die Verbindung zur Serverappliance verlieren. Zur Wiederherstellung der Verbindung muss der Benutzer entweder die neue IP-Adresse verwenden oder die Aktualisierung des DNS-Servers abwarten.

So legen Sie die IP-Einstellungen auf der Registerkarte "Allgemein" fest oder ändern sie

  1. Wählen Sie auf der primären Navigationsleiste Netzwerk aus.
  2. Wählen Sie Schnittstellen aus.
  3. Wählen Sie die zu ändernde Netzwerkverbindung aus.
  4. Wählen Sie in der Liste Tasks den Eintrag IP aus.
  5. Wählen Sie die Registerkarte Allgemein aus.
  6. Geben Sie an, ob die Konfiguration automatisch vom DHCP-Server abgerufen wird oder ob Sie die IP-Adressen statisch konfigurieren.
  7. Wenn Sie die Konfiguration vom DHCP-Server erhalten möchten, klicken Sie auf OK.
  8. Bei Verwendung der statischen IP-Einstellungen geben Sie die IP-Adresse, die Subnetzmaske und das Standardgateway in den bereitgestellten Feldern ein.

So legen Sie die IP-Einstellungen auf der Registerkarte "Fortgeschritten" fest oder ändern sie

  1. Wählen Sie auf der primären Navigationsleiste Netzwerk aus.
  2. Wählen Sie Schnittstellen aus.
  3. Wählen Sie die zu ändernde Netzwerkverbindung aus.
  4. Wählen Sie in der Liste Tasks den Eintrag IP aus.
  5. Wählen Sie die Registerkarte Fortgeschritten aus.
  6. Geben Sie im Feld IP-Adresse auf der rechten Seite die IP-Adresse ein, und klicken Sie dann auf Hinzufügen.
  7. Verfügen Sie über eine LAN-Verbindung, geben Sie die entsprechenden Informationen im Feld Subnetzmaske ein.

    Eine Subnetzmaske ist eine 32-Bit-Zahl, die aus vier Zahlen zwischen 0 und 255 gebildet wird, die durch Punkte voneinander getrennt sind. In der Regel werden in einer Subnetzmaske die Standardzahlen 0 und 255 verwendet, z. B. 255.255.255.0. Sie kann jedoch auch andere Zahlenwerte enthalten, wodurch die Konfiguration des Subnetzes für ein einzelnes TCP/IP-Netzwerk angegeben wird. Diese Zahl, deren Wert nicht 0 oder 255 beträgt, wird mit der IP-Adressennummer kombiniert, um das Netzwerk des Computers zu identifizieren.

  8. Aktualisieren Sie gegebenenfalls die Angabe unter IP-Verbindung metrisch.

    Die Metrik gibt die Kosten für die Verwendung der zu dieser Verbindung gehörenden Routen an und wird als Wert dieser Routen in der Spalte Metrik in der IP-Routingtabelle angezeigt. Liegen für ein Ziel mehrere Routen vor, wird die Route mit der niedrigsten Metrik verwendet. Der Standardwert ist 1.

  9. Wiederholen Sie Schritte 1 bis 3 für jede weitere hinzuzufügende IP-Adresse.

So legen Sie die Einstellungen der Gateway-Adresse fest oder ändern sie

  1. Geben Sie in den Feldern Gateway-Adresse und Metrisch die IP-Adresse des Standardgateways und die Metrik ein, und klicken Sie dann auf Hinzufügen.
  2. Wiederholen Sie Schritt 1 für jedes hinzuzufügende Standardgateway.
  3. Klicken Sie nach Änderung der Konfigurationseinstellungen in diesem Bildschirm auf OK.
