Vorheriges Thema

Verwalten lokaler Gruppen

Bei einem lokalen Benutzer- oder Gruppenkonto handelt es sich um ein Konto, das auf der Serverappliance selbst vorhanden ist und Benutzern oder Gruppen Zugriff auf die zugehörigen Ressourcen gewährt. Die Serverappliance kann ebenfalls so konfiguriert werden, dass die Zugriffsrechte Domänenbenutzern und -gruppen erteilt werden. Domänenbenutzer und -gruppen werden in einer Microsoft® Windows NT® 4- oder Microsoft® Active Directory™-Domäne verwendet. Sie können lokale Benutzer sowie Domänenbenutzer und -gruppen zu lokalen Gruppen hinzufügen.

Benutzer und Gruppen sind für die Sicherheit unter Microsoft Windows von Bedeutung, da Sie die Ausführung bestimmter Aktionen seitens der Benutzer und Gruppen durch Zuweisen von Berechtigungen einschränken können. Eine Berechtigung ist eine einem Objekt (normalerweise eine Datei, ein Ordner oder eine Freigabe) zugewiesene Regel, die festlegt, welche Benutzer auf welche Weise Zugriff auf das Objekt haben. Jeder Domänen- oder lokale Benutzer, der zur lokalen Administratorengruppe auf der Serverappliance gehört, verfügt über Administratorrechte für die Serverappliance. Entsprechend weist jedes Mitglied einer Gruppe, die der Administratorgruppe auf dem lokalen Computer zugewiesen wurde, Administratorrechte für den jeweiligen Computer auf. Beispielsweise könnten Sie die Gruppe Teamleitung, die aus Thomas, Maria, Hanna und Jörg besteht, der Administratorgruppe auf der Serverappliance zuweisen. Jedem Gruppenmitglied würden dann Administratorrechte für die Serverappliance zugewiesen.
