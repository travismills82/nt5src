Administrar los grupos locales

Una cuenta de usuario o de grupo local es una cuenta que existe en el propio módulo servidor y que concede a los usuarios o los grupos acceso a sus recursos. El módulo servidor también puede configurarse para que conceda acceso a usuarios y grupos del dominio. Los usuarios y los grupos del dominio son aquellos que existen en un dominio de Microsoft® Windows NT® 4 o Microsoft® Active Directory™. Puede agregar usuarios locales, usuarios del dominio y grupos del dominio a los grupos locales.

Los usuarios y los grupos son importantes en la seguridad de Microsoft Windows Powered, ya que puede limitar la capacidad de los usuarios y los grupos para realizar determinadas acciones mediante la asignación de permisos. Un permiso es una regla asociada con un objeto, normalmente un archivo, una carpeta o un recurso compartido, que regula qué usuarios pueden tener acceso al objeto y de qué manera. Cualquier usuario local o del dominio que sea miembro del grupo local Administradores del módulo servidor tiene privilegios administrativos para el módulo servidor. Del mismo modo, cualquier miembro de un grupo que se haya asignado al grupo Administradores del equipo local tiene privilegios administrativos para dicho equipo. Por ejemplo, podría asignar el grupo JefesDeEquipo, que está compuesto por Tomás, María, Elena y Jaime, al grupo administrativo del módulo servidor. Cada uno de estos miembros del grupo JefesDeEquipo tendría así privilegios administrativos en el módulo servidor.
