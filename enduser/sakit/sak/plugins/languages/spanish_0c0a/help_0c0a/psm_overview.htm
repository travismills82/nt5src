Persistent Storage Manager

The Persistent Storage Manager allows you to create snapshot images of volumes on the server appliance. These snapshots, called persistent images, preserve data on selected volumes in case of a system or power failure. Each persistent image is saved as a volume on the file system to allow clients read-only or read/write permission. You can create a persistent image immediately through this configuration system or schedule it as a one-time or recurring event.

Once created, a persistent image of a volume appears as a directory on the original volume.  The image inherits permission rights from the original volume. Images are used the same as conventional system volumes. However, unlike conventional volumes, persistent images can be restored to the precise content of the original volume at the time the snapshot was created.

Persistent Storage Manager includes a fully integrated system backup utility. This utility allows the creation of system volume backups to any local, network attached, or removable device. Recovery is performed using a single system generated bootable floppy diskette. Recovery software provides menu driven or fully automatic recovery for head-less operation.

The Persistent Storage Manager and Disaster Recovery System is fully integrated with Microsoft® Windows® Scheduler, allowing complete unattended management of persistent and disaster recovery image creation and rotation on a periodic basis.

Use the Persistent Storage Manager to control system resource usage, optimization, and persistent image management.
