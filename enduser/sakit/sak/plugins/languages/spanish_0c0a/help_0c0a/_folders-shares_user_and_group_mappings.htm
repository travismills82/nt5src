Asignaciones de usuarios y grupos

Para proporcionar seguridad a los archivos del módulo servidor a los que se tiene acceso desde entornos UNIX, el protocolo NFS requiere que el administrador del sistema asigne cuentas de usuario o de grupo de UNIX a unas cuentas gemelas en el módulo servidor. Los usuarios tienen así derechos de acceso desde UNIX equivalentes a los de Microsoft Windows. Como alternativa, los sitios Web que tienen necesidades de seguridad menores pueden eludir este procedimiento de asignación y tratar a todos los usuarios de UNIX como usuarios anónimos.

Asignaciones de usuarios y grupos le permite crear asignaciones entre cuentas de usuario y grupo de Windows y de UNIX incluso aunque los nombres de usuario y de grupo en ambos entornos no sean idénticos. Puede que lo más importante sea que Asignaciones de usuarios y grupos le permite mantener una única base de datos de asignaciones para toda la empresa.

Puede utilizar asignaciones simples, que asignan cuentas de Windows y de UNIX que tengan nombres idénticos. También puede crear asignaciones avanzadas para asociar cuentas de Windows y de UNIX que tengan nombres diferentes, que puede utilizar junto con las asignaciones simples. Aún más, con Asignaciones de usuarios y grupos puede obtener la información de usuario, contraseña y grupo de UNIX desde uno o varios servidores del protocolo de información de red (NIS), o desde archivos de contraseñas y grupos importados.
