Resolución de nombres DNS

Para que otras personas puedan llegar a su sitio Web en una intranet o desde Internet, debe disponer de una dirección IP única que identifique a su equipo en la red. Esta dirección consiste en una larga cadena de números separados por puntos, por ejemplo 172.16.255.255. Como las direcciones numéricas son difíciles de recordar, se utilizan nombres de texto o nombres descriptivos, para proporcionar a los visitantes una dirección fácil de recordar, como www.microsoft.com. La resolución de nombres consiste en proporcionar la dirección numérica correcta a partir del nombre descriptivo que se escribe en el explorador cliente.

Sistemas de resolución de nombres

Los componentes de red de Windows se basan en la convención de nomenclatura de NetBIOS. Por el contrario, los componentes de TCP/IP se basan en una convención de nomenclatura conocida como Sistema de nombres de dominio (DNS). En Windows, el nombre de host DNS predeterminado del módulo servidor es el mismo que el nombre de equipo NetBIOS. La asignación de nombres de equipos con direcciones IP se puede conseguir mediante uno de los dos métodos siguientes:

  • Estática: el administrador del sistema crea un archivo de texto con los nombres DNS, llamado archivo HOSTS, o un archivo LMHOSTS para los nombres NetBIOS, y escribe el nombre y la dirección IP de cada equipo. A continuación se distribuye el archivo por la red. Cuando se solicita una conexión con otro equipo, el archivo se utiliza para resolver el nombre a la dirección IP correcta. Este sistema funciona bien en redes sencillas que cambian con poca frecuencia.
  • Dinámica: cuando un equipo cliente se conecta a una red que tiene un servidor DHCP, el servidor DHCP asigna una dirección y envía la asignación de dirección IP a un servidor de Servicio de nombres Internet de Windows (WINS). El servidor WINS registra el nombre del equipo y cuando se solicita un nombre de equipo NetBIOS, el servidor WINS resuelve el nombre a la dirección IP correcta. El reconocimiento y la asignación automáticos de nombres y direcciones de equipos facilita la labor administrativa en redes grandes o que cambian con mucha frecuencia.

Los nombres DNS se suelen resolver utilizando información estática. El servidor DNS contiene una parte de la base de datos estática que enumera nombres y direcciones de hosts. Si el nombre solicitado no está en la parte de la base de datos del servidor DNS, envía la consulta a otros servidores DNS para conseguir la información solicitada. Los servidores DNS que se ejecutan en Windows pueden configurarse para que consulten servidores WINS para la resolución de nombres de los niveles inferiores de la estructura jerárquica de nomenclatura DNS. Como WINS asigna los nombres de los equipos de forma dinámica, esta opción convierte DNS de un sistema estático a un sistema dinámico.
