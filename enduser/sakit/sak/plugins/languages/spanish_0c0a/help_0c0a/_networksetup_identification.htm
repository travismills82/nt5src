Identificación

El módulo servidor debe tener un nombre. Los equipos cliente utilizan este nombre para tener acceso a los recursos compartidos de archivos que residen en este módulo servidor.

El módulo servidor se puede configurar como miembro de uno de los grupos siguientes:

  • Dominio de Microsoft® Windows NT® 4
  • Dominio de Microsoft Active Directory™
  • Grupo de trabajo

    Si no existen grupos de trabajo en la red, por ejemplo si se trata de un entorno UNIX, se debe seleccionar la opción de grupos de trabajo y utilizar cualquier nombre arbitrario. 

También se pueden crear cuentas de usuario localmente en el módulo servidor; sin embargo, el uso de un dominio o un directorio elimina la necesidad de crear cuentas de usuario locales para todos los usuarios del módulo servidor

Una práctica recomendada después de unirse a un dominio consiste en agregar uno o varios usuarios de dominio al grupo local de administradores y, a continuación, volver a conectar con dichos nombres de usuario para administrar el módulo servidor.

Para establecer el nombre del módulo servidor y el dominio al que pertenece

  1. En la barra de exploración principal, elija Red.
  2. Elija Identificación.
  3. En los cuadros proporcionados para ello, escriba el nombre del módulo servidor y el sufijo del Sistema de nombres de dominio (DNS) apropiados. El sufijo DNS se anexa al nombre de host para crear el nombre completo de máquina.

    Nota   El sufijo DNS es opcional; sin embargo, si desea establecer o cambiar el nombre del módulo servidor, éste es el único cuadro en el que puede hacerlo.

  4. Seleccione si el equipo cliente formará parte de un grupo de trabajo o de un dominio.
  5. Si la máquina va a formar parte de un dominio, escriba el nombre de usuario y la contraseña de la persona que dispone de los permisos para agregar equipos cliente al dominio.
  6. Si el módulo servidor es accesible para clientes AppleTalk o NetWare, escriba el nombre de AppleTalk o NetWare en el cuadro apropiado.

    De forma predeterminada, el nombre AppleTalk del módulo servidor será el mismo que el nombre del módulo servidor normal; sin embargo, el nombre NetWare debe ser diferente del nombre del módulo servidor. De forma predeterminada, el nombre NetWare será el nombre del módulo servidor seguido de _NW.

    Todos los cambios realizados en el nombre del módulo servidor actualizarán también los nombres NetWare y AppleTalk.

  7. Haga clic en Aceptar.
  8. Cuando se le pregunte si desea reiniciar el módulo servidor, puede aceptar o cancelar el reinicio.
    • Si elige Aceptar, el módulo servidor se reiniciará y aparecerá la página Reiniciando. Cuando el módulo servidor se haya reiniciado, aparecerá la Página principal de la interfaz de usuario (UI) Web y las modificaciones surtirán efecto.
    • Si elige Cancelar, las modificaciones realizadas en la identidad del módulo servidor no surtirán efecto hasta el siguiente reinicio.
