Hosts TCP/IP

Los componentes de red de Windows se basan en la convención de nomenclatura de NetBIOS. Por el contrario, los componentes de TCP/IP se basan en una convención de nomenclatura conocida como Sistema de nombres de dominio (DNS). En Windows, el nombre de host DNS predeterminado es el mismo que el nombre de equipo NetBIOS. La asignación de nombres de equipos con direcciones IP se puede conseguir mediante uno de los dos métodos siguientes:

  • Estática: el administrador del sistema crea un archivo de texto con los nombres DNS, llamado archivo HOSTS, o un archivo LMHOSTS para los nombres NetBIOS, y escribe el nombre y la dirección IP de cada equipo. A continuación se distribuye el archivo por la red. Cuando se solicita una conexión con otro equipo, el archivo se utiliza para resolver el nombre con la dirección IP correcta. Este sistema funciona bien en redes sencillas que cambian con poca frecuencia.
  • Dinámica: cuando un equipo cliente inicia sesión, un servidor DHCP asigna una dirección y envía la asignación de dirección IP a un servidor de Servicio de nombres Internet de Windows (WINS). El servidor WINS registra el nombre del equipo y cuando se solicita un nombre de equipo NetBIOS, el servidor WINS resuelve el nombre a la dirección IP correcta. El reconocimiento y la asignación automáticos de nombres y direcciones de equipos facilita la labor administrativa en redes grandes o que cambian con mucha frecuencia.

Los nombres DNS se suelen resolver utilizando información estática. El servidor DNS contiene una parte de la base de datos estática que enumera nombres y direcciones de hosts. Si el nombre solicitado no está en la parte de la base de datos del servidor DNS, envía la consulta a otros servidores DNS para conseguir la información solicitada. Los servidores DNS que se ejecutan en Windows pueden configurarse para que consulten servidores WINS para la resolución de nombres de los niveles inferiores de la estructura jerárquica de nomenclatura DNS. Como WINS asigna los nombres de los equipos de forma dinámica, esta opción convierte DNS de un sistema estático a un sistema dinámico.

Si va a configurar varios sitios Web o FTP en un mismo servidor, cada uno con su propia dirección IP, podría tener problemas con el registro automático en DNS. Para asegurar el enlace IP y el registro DNS correctos, deshabilite el registro DNS automático en Windows 2000 Server y configure manualmente el registro DNS para los sitios Web. Para obtener más información acerca de cómo deshabilitar el registro DNS automático y cómo configurar manualmente el registro DNS, consulte la documentación de Windows 2000 Server.

Si desea asignar varios nombres a un módulo servidor, debe utilizar una asignación de nombres estática para el módulo servidor. En un equipo puede asignar varios nombres con una dirección IP o bien puede utilizar varios nombres, cada uno de ellos asignado a su propia dirección IP.

Para modificar el archivo Hosts

  1. En la página Red, elija Configuración global.
  2. Seleccione la ficha Hosts TCP/IP. De forma predeterminada, el cuadro Archivo Hosts contiene la configuración actual del archivo Hosts.
  3. Modifique el archivo Hosts; para ello, haga clic en el cuadro y modifique la información deseada.
  4. Haga clic en Aceptar.
