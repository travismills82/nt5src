Directory Quota Alarm Notification

Use the Notification area of the Warning Alarm and Severe Alarm tabs to specify the message to send when an alarm is activated and where you want the message sent. The following controls appear in the Notification area of the Severe Alarm and Warning Alarm tabs.

  • Insert macro A set of predefined message macros are available for alarm notification messages. All macros must be surrounded by brackets and must be uppercase.
    [USER]  Current user that caused the action.
    [USER NO DOMAIN] User without the domain name included.
    [OBJECT NAME]  The directory name that caused the action.
    [OBJECT NAME SHARE] Translate the object name to a share if possible. If multiple shares point to an object, the first share found will be used.
    [PERCENT FREE] Percentage of the directory quota that is unused.
    [PERCENT USED] Percentage of the directory quota that is used.
    [SERVER NAME] The name of the appliance.
    [CURRENT USAGE] Directory space used in bytes.
    [CURRENT USAGE KB] Directory space used in kilobytes.
    [CURRENT USAGE MB] Directory space used in megabytes.
    [SPACE LIMIT]  Space limit in bytes.
    [SPACE LIMIT KB] Space limit in kilobytes.
    [SPACE LIMIT MB] Space limit in megabytes
  • Message The message field can contain any message and a set of predefined macros. The maximum length of the message is 1024 characters. The application may send a pop-up message to the user when the alarm activates. When the message field is used, all macros are translated to the appropriate values and the message is passed to NET SEND for delivery.
  • Notify user Sends a pop-up alarm message to the user.
  • Send to event log Sends the alarm message to the server appliance's event log.
