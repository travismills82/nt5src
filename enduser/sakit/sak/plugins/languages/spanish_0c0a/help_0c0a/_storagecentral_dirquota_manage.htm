Managing a Directory Quota

When you define a directory quota on a partition or directory, users with read/write access to the directory who have not exceeded their quota limit can store data in the directory.  Users are notified of disk space usage with two alarms: warning and severe.  

Use caution when placing a directory quota on the system partition. If the system does not have sufficient quota to write files, it may fail. Also, if the system partition does not have enough space to write temporary files during boot, the system may not restart. You can avoid this by using caution when placing quotas on the system partition or system directories.

To manage directory quotas

  1. On the primary navigation bar, choose Disks.
  2. Choose Directory Quota.

You can place a limit on the partition as well, which will enforce a storage resource management policy on the drive. This page contains the following information for each directory quota:

  • Directory Name The name of the directory that contains the quota.
  • % Used The percentage of space limit used by the directory.  The display units are automatically calculated.
  • Space Limit The space limit on the directory. The display units are automatically calculated.
  • Space Used The current amount of space being used by the directory.  The display units are automatically calculated.
  • Space Free The amount of free space available in the directory.  This is the Space Limit value minus the Space Used value.
  • Highwater  The highest amount of space used by the directory.
