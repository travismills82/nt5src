File Screening Alarm Settings

The Alarm tab of the file screening feature includes the following two settings:

  • Active Enforces the screening policy but alarms continue to activate if a screening policy is violated.
  • Record alarm in database Records the screening activity in the audit database.  The monitor task views this database and includes passive policies. File screening uses an ODBC database to audit file creation activity. The database may be stored on any server depending on your configuration. By default, the database is stored on the server appliance in Microsoft® Access. You may create a centralized database for your appliance by changing the path on the Preferences page.
