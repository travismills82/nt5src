Establecer mensaje de correo electrónico de alerta

Es posible configurar el módulo servidor para que genere una notificación automática por correo electrónico cuando se produzca una alerta. Puede elegir entre recibir notificaciones para todos los tipos de alertas o sólo para determinados tipos de alertas, como informativas, advertencias o críticas.

Esta característica utiliza el servicio SMTP del módulo servidor para enviar correo electrónico. En un entorno normal de Internet, no es necesario configurar una puerta de enlace SMTP. Sin embargo, para enviar correo electrónico a Microsoft® Exchange Server o a Lotus Notes, tiene que proporcionar el nombre de la puerta de enlace SMTP específica. Debe incluir el nombre del servidor de puerta de enlace SMTP, o la dirección IP, en el campo del servidor SMTP de la interfaz de usuario (UI) de Web. Póngase en contacto con el administrador de Microsoft Exchange para conocer el nombre del servidor de puerta de enlace SMTP.

Para establecer la característica de alerta por correo electrónico

  1. En la barra de exploración principal, elija Mantenimiento.
  2. Elija Correo electrónico de alerta.
  3. Haga clic en el botón Deshabilitar el mensaje de correo electrónico de alerta.

    - O bien -

    1. Haga clic en el botón Habilitar el mensaje de correo electrónico de alerta y, a continuación, active las casillas de verificación adecuadas para las circunstancias bajo las cuales desea enviar correo electrónico de alerta.
    2. En el cuadro Para, escriba la dirección de correo electrónico del administrador del sistema.

      Para que el mensaje de correo electrónico de alerta se envíe a varias direcciones, escriba las direcciones deseadas en el cuadro Para, separadas por comas.

    3. En el cuadro Con, escriba el nombre de la puerta de enlace SMTP o la dirección IP del servidor SMTP.
    4. Para probar la configuración, elija Prueba.

      Después de hacer clic en el botón Prueba se enviará un mensaje de correo electrónico de prueba. Si el servicio SMTP no está instalado en el equipo, o si no hay cable de red, recibirá el mensaje de error "No se puede enviar un mensaje de correo electrónico de prueba. "

      Si no recibe el mensaje de correo electrónico de prueba, incluso aunque haya aparecido el mensaje "Se envió un mensaje de correo electrónico de prueba. Compruebe el buzón de correo del administrador" en la interfaz de usuario de Web, la causa más probable del error es el servidor SMTP. Para quitar este error, restablezca el nombre del servidor SMTP. 

    5. Haga clic en Aceptar.
