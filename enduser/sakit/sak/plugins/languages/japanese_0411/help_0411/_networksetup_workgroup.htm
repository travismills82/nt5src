ワークグループ

ワークグループはコンピュータを単純にグループ化したもので、ユーザーがそのグループ内でプリンタや共有フォルダなどを探しやすくする目的で使用されます。サーバー アプライアンスは、ワークグループ、Active Directory 環境、または Windows NT 4 ドメインのメンバにすることができます。Microsoft Windows 2000 のワークグループは、ドメインで提供されているユーザー アカウントの一元管理および認証をサポートしません。

ワークグループには、コンピュータと同じ名前を指定できません。ワークグループ名は 15 文字以内で指定します。また、使用できない文字には ; : " < > * + = \ | ? , があります。

サーバー アプライアンスのワークグループ メンバシップを設定または変更するには

  1. プライマリ ナビゲーション バーで、[ネットワーク] をクリックします。
  2. [識別] をクリックします。
  3. [ワークグループ] をクリックし、参加するワークグループの名前を入力します。
  4. ドメインに属していたサーバー アプライアンスをワークグループに参加させると、ドメインからそのサーバー アプライアンスが離脱し、コンピュータ アカウントが無効になります。
  5. [OK] をクリックします。
  6. サーバー アプライアンスを再起動するかどうかを尋ねられます。

    サーバー アプライアンスを再起動するかどうかを指定します。

    • [OK] をクリックすると、サーバー アプライアンスが再起動され、再起動が進行中であることを示すページが表示されます。サーバー アプライアンスがオンラインに戻ると、Web UI の既定のページが表示され、変更が有効となります。
    • [キャンセル] をクリックした場合は、サーバー アプライアンスの識別情報に対する変更が次に再起動するまで有効となりません。
