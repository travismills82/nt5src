WINS の構成

このプロパティ ページでは、外部 WINS サーバーのアドレスを入力できます。サーバー アプライアンスには WINS サーバーが含まれていません。ここで説明する目的から、サーバー アプライアンスは WINS クライアントとして機能します。

WINS 対応のクライアント コンピュータは、WINS サーバーを直接使用するように構成できます。一般に、ほとんどの WINS クライアント コンピュータには複数の NetBIOS (ネットワーク基本入出力システム) 名が設定され、ネットワークに登録されています。これらの名前は、Messenger や Workstation Service などの各種ネットワーク サービスを公開するために使用されます。各コンピュータは、これらのサービスをさまざまに利用してネットワーク上のほかのコンピュータと通信します。

WINS 対応クライアントが WINS サーバーと通信することで、ユーザーは次のタスクを実行できます。

  • WINS データベースにクライアント名を登録する。
  • WINS データベースを使ってクライアント名を更新する。
  • WINS データベースからクライアント名を解放する。
  • ユーザー名、NetBIOS 名、DNS 名、および IP アドレスについて、WINS データベースからマッピングを取得して名前を解決する。

WINS に対応していないクライアントでも、WINS プロキシを使用することで制限的にこれらの処理に参加できます。DHCP サーバーを使って WINS サーバーに IP アドレスを割り当てている場合は、WINS サーバーのアドレスを追加する必要はありません。

Web UI で操作できる WINS アドレスは 2 つだけです。それもアダプタの IP アドレスを静的に割り当てている場合に限られています。DHCP が有効であれば、既存のアドレスの一方または両方を削除して別のアドレスを追加できますが、DHCP 対応アダプタからすべての WINS サーバーを削除することはできません。WINS アドレスを 2 つとも削除したうえで、アドレスを 1 つも追加しなかった場合は、DHCP によって自動的に WINS アドレスが割り当てられます。

WINS クライアントは、起動時またはネットワークへ参加する際に、それぞれの名前を WINS サーバーに登録しようとします。その後 WINS クライアントは、必要に応じて WINS サーバーに問い合わせ、リモート名を解決します。

サーバー アプライアンスの WINS 設定を変更するには

  1. プライマリ ナビゲーション バーで、[ネットワーク] をクリックします。
  2. [インターフェイス] をクリックします。
  3. 変更するネットワーク接続を選択します。
  4. [タスク] ボックスの一覧で、[WINS] をクリックします。
  5. [WINS サーバー] ボックスの一覧で、削除する WINS サーバーの IP アドレスを選択し、[削除] をクリックします。
  6. [WINS サーバー アドレス] ボックスに、WINS サーバーの IP アドレスを入力し、[追加] をクリックします。
  7. さらに WINS サーバーの IP アドレスを追加する場合は、手順 4 と 5 を繰り返します。
  8. [OK] をクリックします。
