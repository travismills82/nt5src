一般的なアプリケーション プロパティの設定

[アプリケーション設定] ページでは、以下の操作を実行できます。

  • アクセス許可の設定
  • 既定のページの指定
  • Microsoft® FrontPage® 拡張機能の使用の有効化または無効化 (既定は無効)
  • サーバー アプライアンスが管理する Web サイトの FTP Web サイトの作成

アプリケーション設定を設定または変更するには

  1. プライマリ ナビゲーション バーで、[サイト] をクリックします。
  2. 設定を変更する Web サイトを選択します。
  3. [タスク] ボックスの一覧で、[変更] をクリックします。
  4. [アプリケーション設定] タブをクリックします。
  5. [既定の実行アクセス許可] ボックスの一覧で、スクリプトの実行レベルを選択します。
    • なし - サーバーでスクリプトや実行可能ファイルを実行しません。スクリプトには、Active Server Pages (ASP) アプリケーションも含まれます。
    • スクリプトのみ - サーバーで ASP アプリケーションなどのスクリプトのみを実行します。
    • スクリプトおよび実行形式 - サーバーで ASP アプリケーションなどのスクリプトと実行可能ファイルの両方を実行します。
  6. [既定のページ] ボックスの一覧で、適切な既定の Web ページを選択します。
    • 既定のページの一覧にページを追加するには、追加するページのファイル名を [ファイル名] ボックスに入力し、[追加] をクリックします。
    • 既定のページの一覧からページを削除するには、削除するページのファイル名を選択し、[削除] をクリックします。
  7. Microsoft FrontPage の拡張機能を使うと、管理者は FrontPage で提供されるグラフィカル ユーザー インターフェイス (GUI) で Web サイトを表示したり管理することができます。FrontPage を使用してサーバー アプライアンス上の Web サイトを表示および管理する場合は、[Microsoft FrontPage 拡張を許可する] チェック ボックスをオンにします。
  8. [内容を変更するための FTP Web サイトを作成します] チェック ボックスをオンにすると、FTP を使って接続可能な Web サイトを作成し、管理者アカウントからその Web サイトにアクセスしてファイルを変更することができます。
  9. [匿名アクセスを許可] チェック ボックスをオンにすると、リモート ユーザーは、ユーザー名やパスワードを入力せずにサーバー アプライアンスへ接続できるようになります。匿名でログオンしたユーザーには、ゲスト アカウントに割り当てられたアクセス許可のみが与えられます。通常、匿名アクセスはインターネット Web サイトに使用されます。
