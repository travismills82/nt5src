識別

サーバー アプライアンスには名前を付ける必要があります。クライアント コンピュータは、この名前を使ってサーバー アプライアンス上のファイル共有にアクセスします。

サーバー アプライアンスは、次に示すグループのいずれかのメンバとして構成できます。

  • Microsoft® Windows NT® 4 ドメイン
  • Microsoft Active Directory™ ドメイン
  • ワークグループ

    ネットワーク上にワークグループが存在しない場合 (たとえば UNIX 環境) は、ワークグループ オプションを選択し、任意の名前を使用する必要があります。 

サーバー アプライアンス上でユーザー アカウントをローカルに作成することもできますが、ドメインやディレクトリを使用すれば、サーバー アプライアンスのすべてのユーザーに対してローカルのユーザー アカウントを作成する必要がなくなります。

サーバー アプライアンスをドメインに加えた場合は、後で 1 人または複数のドメイン ユーザーをローカル管理者グループに追加し、これらのユーザー名でサーバー アプライアンスを管理することをお勧めします。

サーバー アプライアンスの名前およびドメイン メンバシップを設定するには

  1. プライマリ ナビゲーション バーで、[ネットワーク] をクリックします。
  2. [識別] をクリックします。
  3. 表示されたボックスに、適切なサーバー アプライアンス名と DNS (ドメイン ネーム システム) サフィックスを入力します。DNS サフィックスがホスト名に付加されて、完全修飾されたマシン名が作成されます。

    注   DNS サフィックスの追加は任意ですが、サーバー アプライアンスの名前を設定または変更する場合は、必ずこのボックスを使用します。

  4. クライアント コンピュータをワークグループまたはドメインに加えるかどうかを選択します。
  5. コンピュータをドメインに加える場合は、クライアント コンピュータをドメインに追加する権限を持つユーザーのユーザー名とパスワードを入力します。
  6. AppleTalk クライアントまたは NetWare クライアントからサーバー アプライアンスにアクセスできる場合は、AppleTalk 名または NetWare 名を該当するボックスに入力します。

    既定では、サーバー アプライアンスの AppleTalk 名はサーバー アプライアンスの標準の名前と同じになります。ただし、NetWare 名とサーバー アプライアンス名は異なっていなければなりません。既定では、サーバー アプライアンス名に _NW を付加したものが NetWare 名となります。

    サーバー アプライアンス名を変更すると、NetWare 名と AppleTalk 名も更新されます。

  7. [OK] をクリックします。
  8. サーバー アプライアンスの再起動が指示されるので、再起動するかしないかを選択します。
    • [OK] をクリックすると、サーバー アプライアンスが再起動され、[再起動中] ページが表示されます。サーバー アプライアンスが再起動されると、Web UI (ユーザー インターフェイス) のホーム ページが表示され、変更が有効となります。
    • [キャンセル] をクリックした場合は、サーバー アプライアンスの識別情報に対する変更が次に再起動するまで有効となりません。
