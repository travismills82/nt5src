明示的なグループ マップの構成

[ユーザーとグループのマッピング] を使用すれば、Microsoft Windows 環境と UNIX 環境で同じユーザー名やグループ名が使用されていない場合でも、Windows プラットフォームと UNIX プラットフォーム間でユーザー アカウントやグループ アカウントを関連付けるためのマップを作成できます。

[ユーザーとグループのマッピング] では、Windows ユーザーと UNIX ユーザー/グループ間の 1 対 1 マップを設定することもできます。たとえば、Windows ユーザー名を UNIX ユーザー名に関連付けたり、UNIX グループを 1 つまたは複数の Windows ユーザー アカウントに関連付けることができます。同じユーザーが Windows アカウントと UNIX アカウントで異なるユーザー名を使用している場合も、明示的なマップを使用します。[明示的なグループ マップ] を使用すれば、エンタープライズ全体を 1 つのマッピング データベースで管理できます。

明示的なグループ マップを作成するには

  1. プライマリ ナビゲーション バーで、[共有] をクリックします。
  2. [共有プロトコル] をクリックします。
  3. [NFS プロトコル] をクリックし、[プロパティ] をクリックします。
  4. [ユーザーとグループのマッピング] をクリックします。
  5. [全般] タブに必要な情報を入力します。このとき、NIS ドメインあるいは PCNFS パスワード とグループ ファイルは必ず入力します。[全般] タブへ情報を入力する方法については、「ユーザーとグループのマッピングの構成」 を参照してください。
  6. [明示的なグループ マップ] タブをクリックします。
  7. オプション : [NIS ドメイン] フィールドに、関連付ける NIS サーバーの名前を入力します。
  8. [UNIX グループの一覧の作成] ボタンをクリックして、[UNIX グループ] ボックスにすべての UNIX グループを表示します。
  9. 各グループからグループを 1 つ選択し、[追加] をクリックします。

    [明示的に割り当てられたグループ] ボックスに、関連付けたグループが表示されます。たとえば DOMAIN グループに関連付ける場合は、「DOMAIN\<グループ名>」と入力し、UNIX グループを選択して、[DOMAIN\<グループ エントリ>] ボックスの横にある [追加] ボタンをクリックします。

    注   1 つの Windows ドメインのグループを複数の UNIX ドメインに関連付けることができます。ただし、1 つの UNIX ドメインのグループを複数の Windows ドメインに関連付けることはできません。

いずれか 1 つのマップを特定のグループのプライマリ マップとして設定するには

  1. [明示的に割り当てられたグループ] ボックスの一覧で、マップを選択します。
  2. [プライマリの設定] をクリックします。
  3. [OK] をクリックします。

明示的なグループ マップを削除するには

  1. 「明示的なグループ マップを作成するには」の手順 1 〜 4 に従い、[明示的なグループ マップ] ページを表示します。
  2. [明示的に割り当てられたグループ] ボックスの一覧で、削除するグループ マップを選択します。
  3. [削除] をクリックします。
  4. [OK] をクリックします。
