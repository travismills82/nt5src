NFS ロック

NFS ロックを使用すると、ファイル全体またはファイルの一部に排他的にアクセスすることができます。ファイル ロックは、サーバー アプライアンスとクライアントの両方で実現されます。ファイルがロックされると、そのファイルに対してバッファ キャッシュは使用されず、すべての書き込み要求は直ちにサーバーへ送信されます。

システムに障害が発生すると、サーバー アプライアンスでは、サーバー アプライアンスの再起動時にファイルのロック状態を前の状態に復元しようとします。クライアントに障害が発生した場合、サーバー アプライアンスはファイル ロックを解放します。ただし、クライアントの再起動後、少し時間が経過してからファイル ロックは復元要求されます。

NFS ロックを管理するには

  1. プライマリ ナビゲーション バーで、[共有] をクリックします。
  2. [共有プロトコル] をクリックします。
  3. [NFS プロトコル] をクリックし、[プロパティ] をクリックします。
  4. [ロック] をクリックします。
  5. オプション : [現在のロック] ボックスの一覧で、ロックを解放するクライアントを選択します。
  6. オプション : [待機期間] ボックスに、再起動後にサーバー アプライアンスがクライアントでのファイル ロックを再確立するまで待機する時間を秒単位で入力します。
  7. [OK] をクリックします。
