FTP 匿名アクセス

FTP サーバーへの匿名アクセスを有効にすると、anonymous または ftp (anonymous のシノニム) というユーザー名で接続できるようになります。パスワードは必要ありませんが、代わりに電子メール アドレスを入力するよう求められます。既定では、匿名アクセスが有効になっています。

注   Microsoft Windows Powered ユーザー アカウントからユーザー名 anonymous を使用して FTP サーバーにアクセスすることはできません。ユーザー名 anonymous は、匿名ログオン機能を提供するために FTP サーバーで設定されています。ユーザー名 anonymous 使用してログオンするユーザーには、匿名ログオンの FTP サーバー構成に基づいてアクセス許可が与えられます。

コンピュータに FTP プロトコル ソフトウェアをインストールしたら、このソフトウェアが動作するように構成しなければなりません。FTP サーバー プロトコル設定は、次のいずれかの構成になります。

  • サーバーへの匿名 FTP 接続を一切許可しない。この場合、すべてのユーザーは有効な Windows ユーザー名とパスワードを入力しなければなりません。FTP サーバー プロトコルにこの設定を適用するには、[FTP プロトコルのプロパティ] ページの [匿名アクセスを有効] チェック ボックスをオフにします。
  • 匿名ユーザーと Windows ユーザーの両方が FTP サーバーへ接続できるようにする。この場合、ユーザーは、匿名アクセスまたは Windows ユーザー名とパスワードを使用できます。FTP サーバー プロトコルにこの設定を適用するには、[FTP プロトコルのプロパティ] ページの [匿名アクセスを有効] チェック ボックスのみをオンにします。
  • サーバーへの匿名 FTP アクセスのみを許可する。この場合、ユーザーは、Windows ユーザー名とパスワードを使用して FTP サーバーへアクセスできません。FTP サーバー プロトコルにこの設定を適用するには、[FTP プロトコルのプロパティ] ページで、[匿名アクセスを有効] チェック ボックスと [匿名アクセスだけを許可] チェック ボックスを両方ともオンにします。

匿名アクセスを有効にする場合、FTP サーバーへ匿名アクセスするための Windows ユーザー名とパスワードを指定する必要があります。匿名 FTP 転送が発生すると、このダイアログ ボックスで指定したユーザー名に基づいて、ファイルへのアクセスを許可するかどうかが決定されます。
