TCP/IP ホスト

Windows のネットワーク コンポーネントは、NetBIOS の名前付け規則に準拠しています。一方、TCP/IP コンポーネントは DNS (ドメイン ネーム システム) と呼ばれる名前付け規則に準拠しています。Windows では、DNS ホストに既定で NetBIOS コンピュータと同じ名前が付けられます。コンピュータ名と IP アドレスは、次のいずれかの方法でマッピングすることができます。

  • 静的な方法 ― システム管理者によって、HOSTS ファイルと呼ばれる DNS 名のテキスト ファイル、または LMHOSTS ファイルと呼ばれる NetBIOS 名のテキスト ファイルが作成され、そのファイルに各コンピュータの名前と IP アドレスが入力されます。作成されたファイルは、ネットワーク上に配布されます。別のコンピュータへの接続が要求された場合は、このファイルを使って、入力された名前から正しい IP アドレスが解決されます。このシステムは、変更の少ない単純なネットワークに適しています。
  • 動的な方法 ― クライアント コンピュータがログオンすると、DHCP サーバーによってアドレスが割り当てられ、その IP アドレスの割り当てが WINS (Windows インターネット ネーム サービス) サーバーに送信されます。WINS サーバーはコンピュータの名前を登録し、NetBIOS コンピュータ名に対して要求が行われた場合に、その名前を正しい IP アドレスに解決します。このコンピュータ名とアドレスの自動再認識とマッピングによって、大規模なネットワークや変更の多いネットワークの管理に伴う負担が軽減されます。

一般に、DNS 名は静的な情報を使って解決されます。DNS サーバーには、ホスト名とアドレスの一覧で構成される静的データベースの一部が含まれています。要求された名前が DNS サーバーにあるデータベースの一部にない場合、DNS サーバーはほかの DNS サーバーに問い合わせて、要求された情報を取得します。Windows 上で稼動する DNS サーバーについては、DNS の階層型名前付け構造の低いレベルで行われる名前解決を WINS サーバーに問い合わせるように構成できます。WINS はコンピュータ名を動的に割り当てるため、この構成によって DNS が静的なシステムから動的なシステムへと効果的に変更されます。

単一のサーバーに複数の Web サイトまたは FTP サイトをセットアップし、それぞれに独自の IP アドレスを設定すると、自動 DNS 登録で問題が発生する場合があります。IP バインディングと DNS 登録を正確に行うには、Windows 2000 Server の自動 DNS 登録を無効にし、Web サイトに対して手動で DNS 登録を構成します。自動 DNS 登録の無効化および DNS 登録の手動構成の詳細については、Windows 2000 Server のマニュアルを参照してください。

1 つのサーバー アプライアンスに複数の名前を割り当てる場合は、サーバー アプライアンスの名前を静的に割り当てる必要があります。1 台のコンピュータ上で、1 つの IP アドレスに複数の名前をマッピングしたり、それぞれが独自の IP アドレスにマッピングされている複数の名前を使用することができます。

Hosts ファイルを編集するには

  1. [ネットワーク] ページで、[グローバル設定] をクリックします。
  2. [TCP/IP ホスト] タブをクリックします。[ホスト ファイル] ボックスには、現在の Hosts ファイルの構成が自動的に表示されます。
  3. ボックス内をクリックし、情報を編集することで、Hosts ファイルを変更します。
  4. [OK] をクリックします。
