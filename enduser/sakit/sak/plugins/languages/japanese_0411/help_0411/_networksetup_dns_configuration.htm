DNS の構成

DNS (ドメイン ネーム システム) は、TCP/IP ホストに対して使用される静的な階層型ネーム サービスです。ネットワーク管理者は、ホスト名と IP アドレスの一覧によって DNS を構成します。これによりネットワーク上のユーザーは、DNS に問い合わせて、リモート システムを IP アドレスではなくホスト名で指定することができます。

注   このプロパティ ページの目的は、外部 DNS サーバーのアドレスを入力できるようにすることです。サーバー アプライアンスには DNS サーバーが含まれていません。

たとえば、remotehost というシステムのマッピングが DNS データベースに含まれている場合、DNS で名前を解決するように構成されたワークステーションでは、ping 1.2.3.4 ではなく ping remotehost コマンドを使用できます。DNS ドメインと Microsoft Windows ドメインを混同しないようにしてください。

DNS クライアント サーバー モデルでは、DNS データベースの一部の情報、つまりクライアントにコンピュータ名を提供するための情報を含んでいるサーバーによって、インターネットを介して名前を解決するための問い合わせが行われます。

サーバー アプライアンスが DHCP サーバーから DNS サーバーの情報を自動的に取得するように設定するには

  1. プライマリ ナビゲーション バーで、[ネットワーク] をクリックします。
  2. [インターフェイス] をクリックします。
  3. 変更するネットワーク接続を選択します。
  4. [タスク] ボックスの一覧で、[DNS] をクリックします。
  5. [DHCP サーバーから構成を取得します] をクリックします。
  6. [OK] をクリックします。

サーバー アプライアンスが使用する DNS サーバーを手動で設定するには

  1. [ネットワーク] ページで、[インターフェイス] をクリックします。
  2. 変更するネットワーク接続を選択します。
  3. [タスク] ボックスの一覧で、[DNS] をクリックします。
  4. [手動で構成します] をクリックします。
  5. [追加] ボタンの横のボックスに適切な IP アドレスを入力し、[追加] をクリックします。
  6. さらに DNS サーバーを追加する場合は、手順 5 を繰り返します。
  7. DNS サーバーをすべて追加したら、[OK] をクリックします。

    注   IP アドレスを DHCP から取得するように設定し、DNS を手動で設定した場合は、手動による入力が有効となり、サーバー アプライアンス上のプロパティは自動的に [手動で構成します] に設定されます。ただし、[インターフェイス] ページ上の [オブジェクト/タスク セレクタ] の [現在の構成] 列には、IP アドレスの取得元として DHCP が表示されたままになります。[DNS 設定] プロパティ ページに戻ると、手動の構成が保存されているかどうかを確認できます。
