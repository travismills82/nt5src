DNS 名前解決

イントラネットまたはインターネットに開設した Web サイトをほかの人がアクセスできるようにするには、使用するコンピュータをネットワーク上で識別するための一意の IP アドレスが必要です。このアドレスは、ドットで区切られた一連の数値で構成されます (172.16.255.255 など)。数値のアドレスは覚えにくいため、アクセスする人は、フレンドリ名と呼ばれる覚えやすい文字のアドレスを使用します (www.microsoft.com など)。名前解決とは、クライアント ブラウザに入力されたフレンドリ名から正しい数値アドレスを導き出す処理です。

名前解決システム

Windows のネットワーク コンポーネントは、NetBIOS の名前付け規則に準拠しています。一方、TCP/IP コンポーネントは DNS (ドメイン ネーム システム) と呼ばれる名前付け規則に準拠しています。Windows では、サーバー アプライアンスの DNS ホストに、既定で NetBIOS コンピュータ名と同じ名前が付けられます。コンピュータ名と IP アドレスは、次のいずれかの方法でマッピングすることができます。

  • 静的な方法 ― システム管理者によって、HOSTS ファイルと呼ばれる DNS 名のテキスト ファイル、または LMHOSTS ファイルと呼ばれる NetBIOS 名のテキスト ファイルが作成され、そのファイルに各コンピュータの名前と IP アドレスが入力されます。作成されたファイルは、ネットワーク上に配布されます。別のコンピュータへの接続が要求された場合は、このファイルを使って、入力された名前から正しい IP アドレスが解決されます。このシステムは、変更の少ない単純なネットワークに適しています。
  • 動的な方法 ― DHCP サーバーを備えたネットワークにクライアント コンピュータが接続すると、DHCP サーバーによってアドレスが割り当てられ、その IP アドレスの割り当てが WINS (Windows インターネット ネーム サービス) サーバーに送信されます。WINS サーバーはコンピュータの名前を登録し、NetBIOS コンピュータ名に対して要求が行われた場合に、その名前を正しい IP アドレスに解決します。このコンピュータ名とアドレスの自動再認識とマッピングによって、大規模なネットワークや変更の多いネットワークの管理に伴う負担が軽減されます。

一般に、DNS 名は静的な情報を使って解決されます。DNS サーバーには、ホスト名とアドレスの一覧で構成される静的データベースの一部が含まれています。要求された名前が DNS サーバーにあるデータベースの一部にない場合、DNS サーバーはほかの DNS サーバーに問い合わせて、要求された情報を取得します。Windows 上で稼動する DNS サーバーについては、DNS の階層型名前付け構造の低いレベルで行われる名前解決を WINS サーバーに問い合わせるように構成できます。WINS はコンピュータ名を動的に割り当てるため、この構成によって DNS が静的なシステムから動的なシステムへと効果的に変更されます。
