ドメイン

サーバー アプライアンスは、ワークグループ、Active Directory 環境、または Windows NT 4 ドメインに組み込むことが可能です。Microsoft Windows NT 4 および Microsoft Active Directory 環境におけるドメインは、ネットワークの管理者によって定義された共通のディレクトリ データベースを共有するコンピュータの集合です。

Windows のユーザー情報およびグループ情報に利用される Windows ドメインは、一意の名前を持ち、ドメイン管理者によって一元管理されているユーザー アカウントおよびグループ アカウントへのアクセスを提供します。それぞれのドメインには、独自のセキュリティ ポリシー、およびほかのドメインとのセキュリティ リレーションシップが設定されています。また、各ドメインは、Windows コンピュータ ネットワークにおける単一のセキュリティの境界を表しています。Active Directory は、1 つまたは複数のドメインから構成され、各ドメインには複数の物理的な場所が含まれている可能性があります。

DNS のドメインは、DNS 名前空間におけるツリーまたはサブツリーです。DNS ドメインの名前が Active Directory ドメインと一致する場合がよくありますが、DNS ドメインを Microsoft Windows や Active Directory のネットワーク ドメインと混同しないように注意してください。

Netscape のブラウザから Web サーバー アプライアンスまたは NAS にログインするドメイン ユーザーは、既定で「<ドメイン\ユーザー名>」と入力する必要があります。

ログオンに使用される既定のドメインを設定するには

  1. プライマリ ナビゲーション バーで、[メンテナンス] をクリックします。
  2. [ターミナル サービス] をクリックします。
  3. [ターミナル サービス] にログオンします。
  4. [マイ コンピュータ] をマウスの右ボタンでクリックし、[管理] をクリックします。
  5. インターネット インフォメーション サービス (IIS) を開きます。
  6. [管理者 Web サイト] をマウスの右ボタンでクリックし、[プロパティ] をクリックします。
  7. [ディレクトリ セキュリティ] をクリックします。
  8. [匿名アクセスおよび認証コントロール] で [編集] をクリックします。
  9. [基本認証] で [編集] をクリックします。
  10. 認証に使用する既定のドメイン名を入力します。
