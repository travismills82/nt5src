通知電子メールの設定

警告が発生した場合に自動電子メール通知を生成するようにサーバー アプライアンスを構成することができます。発生した警告をすべて通知するか、あるいは情報、警告、重大など特定の種類の警告のみ通知するかを選択できます。

電子メールは、サーバー アプライアンスの SMTP サービスを使用して送信されます。通常のインターネット環境では、SMTP ゲートウェイを構成する必要はありません。ただし、Microsoft® Exchange Server または Lotus Notes に電子メールを送信する場合は、特定の SMTP ゲートウェイの名前を指定する必要があります。Web ユーザー インターフェイス (UI) の SMTP サーバー フィールドに、SMTP ゲートウェイ サーバー名または IP アドレスを入力してください。SMTP ゲートウェイのサーバー名については、Microsoft Exchange の管理者に問い合わせてください。

通知電子メール機能を設定するには

  1. プライマリ ナビゲーション バーで、[メンテナンス] をクリックします。
  2. [通知電子メール] を選択します。
  3. [通知電子メールを無効にする] をクリックします。

    または

    1. [通知電子メールを有効にする] をクリックし、通知電子メールを送信する環境について適切なチェック ボックスを選択します。
    2. [宛先] ボックスに、システム管理者の電子メール アドレスを入力します。

      複数のアドレスに通知電子メールを送信するには、[宛先] ボックスにカンマで区切ってアドレスを入力します。

    3. [使用する SMTP サーバー] ボックスに、SMTP ゲートウェイ名または SMTP サーバーの IP アドレスを入力します。
    4. 設定を試すには、[テスト] をクリックします。

      [テスト] ボタンをクリックすると、テスト用の電子メールが送信されます。SMTP サービスがコンピュータにインストールされていないか、ネットワーク ケーブルが接続されていない場合は、"テスト用の電子メールを送信できません。" というエラー メッセージが表示されます。

      Web UI に "テスト用電子メールは送信されました。管理者の受信トレイを調べてください。" というメッセージが表示されてもテスト用電子メールが全く受信されていない場合、エラーの原因はおそらく SMTP サーバーにあります。このエラーを解決するには、SMTP サーバー名を設定し直してください。 

    5. [OK] をクリックします。
