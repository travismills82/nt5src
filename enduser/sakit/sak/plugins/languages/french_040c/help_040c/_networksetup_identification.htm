Identification

Il faut donner un nom au système serveur. Les ordinateurs clients se servent de ce nom pour accéder aux partages de fichiers qui résident sur ce système serveur.

Le système serveur peut être configuré comme membre de l'un des groupes suivants :

  • Domaine Microsoft® Windows NT® 4
  • Domaine Microsoft Active Directory
  • Groupe de travail

    S'il n'existe aucun groupe de travail sur le réseau, par exemple s'il s'agit d'un environnement UNIX, il convient de sélectionner l'option groupe de travail et d'utiliser n'importe quel nom arbitraire.

Les comptes utilisateurs peuvent également être créés en local sur le système serveur ; cependant, en utilisant un domaine ou un annuaire, vous éliminez le besoin de créer des comptes utilisateurs locaux pour chaque utilisateur du système serveur.

Il est recommandé, après s'être connecté à un domaine, d'ajouter un ou plusieurs utilisateurs du domaine dans le groupe local administrateurs, puis de se reconnecter sous l'un de ces noms afin d'administrer le système serveur.

Pour définir le nom du système serveur et son adhésion au domaine

  1. Dans la barre de navigation principale, choisissez Réseau.
  2. Choisissez Identification.
  3. Dans les zones disponibles, entrez le nom de système serveur approprié et le suffixe DNS. Le suffixe DNS est ajouté à la fin du nom d'hôte pour créer le nom d'ordinateur complet.

    Remarque   L'ajout d'un suffixe DNS est facultatif ; cependant, si vous voulez définir ou changer le nom de votre système serveur, cette zone est la seule dans laquelle vous puissiez le faire.

  4. Choisissez si l'ordinateur client fera partie d'un groupe de travail ou d'un domaine.
  5. S'il doit faire partie d'un domaine, entrez le nom d'utilisateur et le mot de passe de la personne qui détient l'autorisation d'ajouter des ordinateurs clients dans ce domaine.
  6. Si le système serveur est accessible à des clients AppleTalk ou NetWare, entrez le nom AppleTalk ou NetWare dans la zone appropriée.

    Par défaut, le nom AppleTalk de votre système serveur sera le même que son nom standard , en revanche, le nom NetWare doit être différent du nom du système serveur. Par défaut, le nom NetWare sera le nom du système serveur suivi de _NW.

    Toute modification apportée au nom du système serveur met également à jour les noms NetWare et AppleTalk.

  7. Cliquez sur OK.
  8. Lorsque vous êtes invité à redémarrer votre système serveur, vous pouvez soit accepter soit refuser ce redémarrage.
    • Si vous cliquez sur OK, le système serveur redémarre et la page Redémarrage en cours apparaît. Après redémarrage du système serveur, la page d'accueil de l'interface Web s'affiche, et vos modifications ont été prises en compte.
    • Si vous cliquez sur Annuler, les modifications à l'identité du système serveur ne sont pas prises en compte tant que vous n'avez pas redémarré.
