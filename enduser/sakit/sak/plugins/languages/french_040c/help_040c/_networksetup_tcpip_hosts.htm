Hôtes TCP/IP

Les composants réseau de Windows adoptent la convention d'affectation de noms NetBIOS. En revanche, les composants TCP/IP adoptent une convention d'affectation de noms baptisée Domain Name System (DNS). Sous Windows, le nom d'hôte DNS est, par défaut, identique au nom d'ordinateur dans NetBIOS. Le mappage des noms d'ordinateurs sur les adresses IP peut être effectué à l'aide de l'une de ces deux méthodes :

  • Statique—L'administrateur système crée soit un fichier texte pour les noms DNS, nommé fichier HOSTS, soit un fichier LMHOSTS pour les noms NetBIOS, et entre le nom et l'adresse IP de chaque ordinateur. Le fichier est alors distribué sur le réseau. Lorsqu'une requête de connexion à un autre ordinateur est déposée, ce fichier est utilisé pour rechercher la correspondance du nom avec la bonne adresse IP. Ce système fonctionne bien pour les réseaux simples qui changent peu souvent.
  • Dynamique—Lorsqu'un ordinateur client se connecte, un serveur DHCP lui assigne une adresse et envoie l'assignement d'adresse IP à un serveur WINS (Windows Internet Name Service). Le serveur WINS enregistre le nom de l'ordinateur, et lorsqu'une requête de nom d'ordinateur NetBIOS est déposée, le serveur WINS traduit le nom en l'adresse IP correspondante. Cette reconnaissance et ce mappage automatiques des noms et adresses des ordinateurs allègent la charge d'administration des réseaux volumineux ou qui changent souvent.

Les noms DNS sont en général résolus par la méthode statique. Le serveur DNS contient une partie de la base de données statique donnant la liste des noms et des adresses d'hôtes. Si le nom demandé ne se trouve pas dans la portion de base de données du serveur DNS, celui-ci envoie une requête à d'autres serveurs DNS pour obtenir l'information demandée. Le serveur DNS sous Windows peut être configuré pour demander à un serveur WINS la résolution de noms des niveaux inférieurs de la structure d'affectation de noms DNS hiérarchisée. WINS assignant les noms d'ordinateurs de façon dynamique, cela revient à transformer le système DNS statique en un système dynamique.

Si vous voulez configurer plusieurs sites Web ou FTP sur un seul serveur, chacun avec sa propre adresse, vous risquez de rencontrer des problèmes avec l'inscription automatique DNS. Pour vous assurer d'une liaison IP et d'une inscription DNS correctes, désactivez l'inscription DNS automatique de Windows 2000 et configurez manuellement l'inscription DNS pour les sites Web. Pour plus d'informations sur la désactivation de l'inscription DNS automatique et la configuration manuelle de l'inscription DNS, consultez la documentation de Windows 2000 Server.

Si vous souhaitez assigner plusieurs noms à un système serveur, vous devez utiliser un assignement de nom statique pour ce système serveur. Sur un ordinateur, vous pouvez mapper plusieurs noms sur une seule adresse IP, ou vous pouvez utiliser plusieurs noms, chacun étant mappé sur sa propre adresse IP.

Pour modifier le fichier d'hôtes

  1. Dans la page Réseau, choisissez Paramètres globaux.
  2. Sélectionnez l'onglet Hôtes TCP/IP. Par défaut, la zone Fichier d'hôtes contient la configuration actuelle du fichier d'hôtes.
  3. Pour changer le fichier d'hôtes, cliquez dans cette zone et modifiez les informations.
  4. Cliquez sur OK.
