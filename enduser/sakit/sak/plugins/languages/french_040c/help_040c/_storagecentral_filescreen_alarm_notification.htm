File Screening Alarm Notification

Use the Notification area of the Severe Alarm and Warning Alarm tabs to specify the message to send when an alarm is activated and where you want the message sent. The following controls appear in the Notification area of the Severe Alarm and Warning Alarm tabs.

  • Insert macros A set of predefined message macros are available for alarm notification messages. All macros must be surrounded by brackets and must be uppercase.
[OBJECT NAME]  The directory name that caused the action.
[OBJECT NAME SHARE] Translate the object name to a share if possible. If multiple shares point to an object, the first share found will be used.
[FILE NAME] Name of the file that was screened.
[POLICY NAME] The name of the policy that was activated.
[SCREENING GROUP] The screening group that contains the file type that was screened.
[SERVER NAME] The name of the appliance.
[USER]  Current user that caused the action.
[USER NO DOMAIN]  User without the domain name included.
  • Message The message field can contain any message and a set of predefined macros.  The maximum length of the message is 1024 characters. The application may send a pop-up message to the user when the alarm activates. 
  • Notify user Sends a pop-up alarm message to the user.
  • Send to event log Sends the alarm message to the server appliance's event log.
