NFS

Avec le protocole NFS, un système serveur peut se comporter comme un serveur NFS (Network File System). Les utilisateurs peuvent alors partager leurs fichiers dans un environnement hétérogène d'ordinateurs, de systèmes d'exploitation et de réseaux.

Les utilisateurs équipés d'ordinateurs exécutant un logiciel client NFS peuvent accéder aux partages du système serveur en connectant, ou en montant, ces partages sur leur ordinateur.

Les ordinateurs UNIX utilisent une forme de verrouillage appelée Advisory Locking (verrouillage recommandé) pour toutes les demandes de verrouillage. Ceci signifie que le système d'exploitation n'impose pas de verrouillages sur un fichier, et que les applications qui vérifient l'existence de verrous peuvent utiliser ces verrous efficacement. Cependant, le protocole NFS met en œuvre une forme de verrouillage appelée Mandatory Locking (verrouillage obligatoire) même pour les demandes de verrouillage reçues par l'intermédiaire de NFS. Ceci garantit que les verrous acquis via NFS sont visibles dans le protocole Server Message Block (SMB) ainsi que par les applications qui accèdent aux fichiers en local. Le système d'exploitation impose les verrous obligatoires.
