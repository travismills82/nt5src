File Screening

File screening prevents unwanted files from being saved on your server appliance.   Not only do these files increase backup times and consume valuable disk space, they can also put your company at risk of violating piracy and discrimination laws.

The proliferation of Internet communication in the workplace has allowed users to download questionable files to a server appliance. File screening can prevent files containing objectionable graphic images, games, or video from polluting your server appliance. File screening provides a fast and easy way to prevent unwanted file types from using valuable corporate server storage.

File screening includes the following features:

  • Active and passive file screening on directories
  • Best practice file screening policies
  • Notification alarm when file screening policy is violated
  • Audit database containing screened files
  • Customized alarm messages
  • Alarm messages to the event log
  • Alarm messages to a user
  • Storage reports when alarm is activated and sent to intranet Web site
  • Custom script when alarm is activated

File screening is organized in a hierarchy with the directory object being the top node. The node under the directory object contains a set of file screening policies.  An object can be any disk allocation and can be associated with any screening policy. The figure below illustrates the hierarchy.

Use caution when placing screening parameters on the system partition. If certain classes of files are screened from the system partition, the operating system may not have the access to save temporary working files. It is a good idea to exclude systems directories from screening. Another option is to create a passive screening policy that allows files to be saved but the file activity to be logged.
