Création des paramètres de l'application

Dans la page Paramètres de l'application, vous pouvez

  • définir les autorisations
  • spécifier la page par défaut
  • activer ou désactiver l'utilisation des extensions de Microsoft® FrontPage® (désactivées par défaut)
  • créer un site Web FTP pour les sites Web que gère votre système serveur

Pour définir ou modifier les paramètres de l'application

  1. Dans la page de navigation principale, choisissez Sites.
  2. Sélectionnez le site Web à modifier.
  3. Dans la liste Tâches, choisissez Modifier.
  4. Cliquez sur l'onglet Paramètres de l'application.
  5. Dans la liste Autorisations d'exécution par défaut, sélectionnez l'un de ces niveaux d'exécution de script :
    • Aucune—N'exécuter aucun script, y compris les applications ASP (Active Server Pages), ni aucun exécutable sur le serveur.
    • Scripts seulement—Exécuter seulement les scripts, tels que les applications ASP, sur le serveur.
    • Scripts et exécutables—Exécuter à la fois les scripts, tels que les applications ASP, et les exécutables sur le serveur.
  6. Dans la liste Page par défaut, sélectionnez la page Web par défaut appropriée.
    • Pour ajouter une page dans la liste des pages par défaut, entrez le nom de fichier de la page à ajouter dans la zone Nom du fichier, puis cliquez sur Ajouter.
    • Pour supprimer une page de la liste des pages par défaut, sélectionnez le nom de fichier de la page à supprimer, puis cliquez sur Supprimer.
  7. Les extensions Microsoft FrontPage permettent aux administrateurs d'afficher et de gérer des sites Web dans l'interface utilisateur graphique que propose FrontPage. Si vous souhaitez afficher et gérer des sites Web sur votre système serveur en utilisant FrontPage, activez la case à cocher Autoriser les extensions Microsoft FrontPage.
  8. Activez la case à cocher Créer un site Web FTP pour modifier le contenu si vous voulez créer un site Web auquel le compte administrateur puisse se connecter avec FTP pour modifier des fichiers.
  9. Activez la case à cocher Autoriser l'accès anonyme pour laisser des utilisateurs distants se connecter à votre système serveur sans nom d'utilisateur ni mot de passe. Les utilisateurs qui ouvrent une session de façon anonyme ne disposent que des autorisations assignées au compte invité. L'accès anonyme est généralement utilisé pour des sites Web Internet.
