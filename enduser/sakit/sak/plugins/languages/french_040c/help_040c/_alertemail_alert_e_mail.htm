Configuration des messages d'alerte

Votre système serveur peut être configuré pour générer un message de notification automatique lorsqu'une alerte se produit. Vous pouvez choisir d'être averti pour tout type d'alerte, ou seulement pour des types spécifiques tels que messages d'information, messages d'avertissement ou messages d'alertes critiques.

Cette fonctionnalité utilise le service SMTP du système serveur pour envoyer le message. Dans un environnement Internet normal, il n'est pas nécessaire de configurer une passerelle SMTP. Cependant, pour envoyer un message électronique à Microsoft® Exchange Server ou à Lotus Notes, vous devez indiquer le nom de la passerelle SMTP appropriée. Vous devez placer le nom de serveur de la passerelle SMTP, ou son adresse IP, dans le champ serveur SMTP de l'interface utilisateur Web. Demandez à votre administrateur Microsoft Exchange le nom de serveur de la passerelle SMTP.

Pour configurer la fonction de message d'alerte

  1. Dans la barre de navigation principale, choisissez Maintenance.
  2. Choisissez Message d'alerte.
  3. Sélectionnez le bouton Désactiver le message d'alerte.

    OU

    1. Sélectionnez le bouton Activer le message d'alerte, puis activez les cases à cocher selon les circonstances dans lesquelles vous souhaitez recevoir des messages d'alerte.
    2. Dans la zone À, entrez l'adresse de messagerie de l'administrateur système.

      Vous pouvez faire envoyer les messages d'alerte à plusieurs adresses ; il vous suffit d'entrer ces adresses dans la zone À, séparées par une virgule.

    3. Dans la zone Avec, entrez le nom de la passerelle SMTP ou l'adresse IP du serveur SMTP.
    4. Pour tester les paramètres, choisissez Tester.

      Lorsque vous cliquez sur le bouton Tester, un message test est envoyé. Si le service SMTP n'est pas installé sur votre ordinateur, ou en l'absence d'un câble réseau, vous recevrez ce message d'erreur : « Impossible d'envoyer le message test. »

      Si vous ne recevez pas ce message test alors que le message « Le message test a été envoyé. Consultez la boîte aux lettres de l'administrateur » s'est affiché dans l'interface Web, l'erreur a très probablement été causée par le serveur SMTP. Pour corriger cette erreur, entrez de nouveau le nom du serveur SMTP. 

    5. Cliquez sur OK.
