Résolution du nom DNS

Pour que les utilisateurs puissent atteindre votre site Web sur un intranet ou sur Internet, vous devez avoir une adresse IP unique identifiant votre ordinateur sur le réseau. Cette adresse est constituée d'une longue liste de chiffres séparés par des points, par exemple 172.16.255.255. Parce qu'une adresse numérique est difficile à mémoriser, des noms texte, baptisés noms conviviaux, sont adoptés pour que les visiteurs disposent d'une adresse facile à mémoriser, telle que www.microsoft.com par exemple. La résolution de noms est la recherche de l'adresse numérique correcte correspondant au nom convivial qui a été entré dans le navigateur d'un client.

Systèmes de résolution de noms

Les composants réseau de Windows adoptent la convention d'affectation de noms NetBIOS. En revanche, les composants TCP/IP adoptent une convention d'affectation de noms baptisée Domain Name System (DNS). Sous Windows, le nom d'hôte DNS de votre système serveur est, par défaut, identique au nom d'ordinateur dans NetBIOS. Le mappage des noms d'ordinateurs sur les adresses IP peut être effectué à l'aide de l'une de ces deux méthodes :

  • Statique—L'administrateur système crée soit un fichier texte pour les noms DNS, nommé fichier HOSTS, soit un fichier LMHOSTS pour les noms NetBIOS, et entre le nom et l'adresse IP de chaque ordinateur. Le fichier est alors distribué sur le réseau. Lorsqu'une requête de connexion à un autre ordinateur est déposée, ce fichier est utilisé pour rechercher la correspondance du nom avec la bonne adresse IP. Ce système fonctionne bien pour les réseaux simples qui changent peu souvent.
  • Dynamique—Lorsqu'un ordinateur client se connecte à un réseau équipé d'un serveur DHCP, celui-ci lui assigne une adresse et envoie l'attribution d'adresse IP à un serveur WINS (Windows Internet Name Service). Le serveur WINS enregistre le nom de l'ordinateur, et lorsqu'une requête de nom d'ordinateur NetBIOS est déposée, le serveur WINS traduit le nom en l'adresse IP correspondante. Cette reconnaissance et ce mappage automatiques des noms et adresses des ordinateurs allègent la charge d'administration des réseaux volumineux ou qui changent souvent.

Les noms DNS sont en général résolus par la méthode statique. Le serveur DNS contient une partie de la base de données statique donnant la liste des noms et des adresses d'hôtes. Si le nom demandé ne se trouve pas dans la portion de base de données du serveur DNS, celui-ci envoie une requête à d'autres serveurs DNS pour obtenir l'information demandée. Le serveur DNS sous Windows peut être configuré pour demander à un serveur WINS la résolution de noms des niveaux inférieurs de la structure DNS hiérarchisée d'affectation de noms. WINS assignant les noms d'ordinateurs de façon dynamique, cela revient à transformer le système DNS statique en un système dynamique.
