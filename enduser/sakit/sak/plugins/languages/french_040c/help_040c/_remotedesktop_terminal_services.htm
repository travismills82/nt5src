Services Terminal Server

Les services Terminal Server vous donnent la possibilité de vous connecter à votre système serveur et de l'administrer à distance, et constituent un moyen de le gérer depuis n'importe quel client. Lorsqu'ils sont installés pour l'administration à distance, deux sessions seulement de services Terminal Server sont autorisées simultanément. Si vous laissez une session ouverte, vous utilisez une licence et vous pouvez gêner d'autres utilisateurs. Si deux sessions sont en cours d'exécution, les nouveaux utilisateurs se verront refuser l'accès.

Remarque Le client avancé des services Terminal Server, disponible depuis la page Maintenance, n'est supporté que par les systèmes équipés d'Internet Explorer.

Fermeture de session et déconnexion

Les services Terminal Server proposent deux options pour fermer un client : vous pouvez soit vous déconnecter, soit fermer la session sur le système.

La déconnexion laisse la session active sur le serveur. Vous pouvez vous reconnecter au serveur et reprendre la session. Si vous exécutez une tâche sur le serveur, par exemple l'exécution d'une requête très longue sur une base de données, vous pouvez démarrer la tâche et vous déconnecter de la session. Plus tard, vous pourrez vous reconnecter au serveur, entrer de nouveau dans la session, puis reprendre la tâche ou contrôler les résultats. Cette option est très utile lorsque vous avez recours à une connexion d'accès distant via un appel longue distance.

Les administrateurs réseau peuvent limiter la durée pendant laquelle une session dont vous êtes déconnecté peut demeurer active sur le serveur. Vérifiez auprès de votre administrateur s'il existe des limites imposées aux sessions.

L'arrêt d'une session s'appelle la fermeture de session. La fermeture de session met fin à l'exécution de la session sur le serveur. Les applications qui s'exécutent au cours de la session sont fermées et les modifications non enregistrées dans les fichiers sont perdues. La prochaine fois que vous vous connectez au serveur, une nouvelle session est créée.

Enfin, le client avancé des services Terminal Server exige que tous les utilisateurs qui se connectent soient authentifiés, et c'est pourquoi les utilisateurs doivent s'identifier chaque fois qu'ils commencent une session.

Pour accéder au client avancé des services Terminal Server

  1. Dans la barre de navigation principale, choisissez Maintenance.
  2. Choisissez Services Terminal Server.
  3. Ouvrez une session.
