Verrous NFS

Les verrous NFS permettent à un processus de bénéficier d'un accès exclusif à tout ou partie d'un fichier. Le verrouillage du fichier est mis en œuvre à la fois sur le système serveur et sur le client. Lorsqu'un fichier est verrouillé, le tampon de mémoire cache n'est pas utilisé pour ce fichier, et chaque demande d'écriture est immédiatement envoyée au serveur.

Après une panne du système, lors du redémarrage du système serveur, celui-ci essaie de restaurer le statut de verrouillage du fichier dans sa condition initiale. Si le client échoue, le système serveur déverrouille le fichier. Mais après avoir redémarré, le client dispose d'un bref délai pour demander le verrouillage du fichier.

Pour gérer les verrous NFS

  1. Dans la barre de navigation principale, choisissez Partages.
  2. Choisissez Protocoles de partage.
  3. Sélectionnez Protocole NFS, puis Propriétés.
  4. Choisissez Verrous.
  5. Facultatif : dans la liste Verrous actuels, sélectionnez le client dont vous voulez supprimer les verrous.
  6. Facultatif : dans la zone Délai d'attente, entrez le nombre de secondes après redémarrage que doit attendre le système serveur avant de rétablir un verrouillage de fichier avec un client.
  7. Cliquez sur OK.
