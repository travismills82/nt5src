Directory Quota Alarm Action

Use the Actions area of an Alarm tab to specify a report or program to run when the alarm is activated. The following actions are available:

  • When the alarm occurs run this report

  • Best Practices This report is a representation of storage management best practices that are a good starting point for disk space analysis. The Best Practice reports give an overall indication of your systems storage health. The report identifies file types, duplicate files, files not backed up, most commonly-used files, and nightly backup capacity requirements.
  • Duplicate Files This report identifies duplicate files that may use an excessive amount of space, which helps you clean up disk space quickly.
  • File Type Summary This report provides usage details for the directory.
  • Files Not Being Backed Up This report identifies files that have been modified but have not been backed up according to the NTFS file system archive bit.This bit is used by backup applications to indicate a file's backup status. This validates file recoverability and protects your data.
  • Large Files This report identifies large files that have not been accessed for 60 days. This is useful to reclaim unused disk capacity, identify archive candidates, and reduce backup and recovery times.
  • Most Commonly Used Files This report displays the amount of raw capacity accessed each day.This is useful for balancing applications, data, and users across available resources, and to prevent underutilized or overburdened file servers.
  • Directory Quota Usage Summary This report identifies all of the directory quotas defined on the appliance and displays the space used, space free, percent used, and highwater mark.
  • When the alarm occurs run this program
  • Executes the specified command in the background. The command may be any valid Windows 2000 command, for example, Mybatchfile.bat.
