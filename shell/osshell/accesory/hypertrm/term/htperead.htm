How HyperTerminal PE is Different from HyperTerminal

On the surface, HyperTerminal PE seems identical to HyperTerminal (except for its banner screen). Both are speedy 32-bit programs that take full advantage of the Windows interface and features, such as TAPI (Telephony API) and Unimodem universal modem support. But looks can be deceiving. Each new version of HyperTerminal PE has added new features not in the versions of HyperTerminal that Microsoft provides, and has corrected problems identified after release of HyperTerminal. Sections below list the new features added in each new version of HyperTerminal PE.

HyperTerminal PE 6.0 Enhancements

  • Addition of VT320 emulation
  • A setting was added to control whether VTxxx emulator host-controlled printing uses the Windows print driver settings or not

HyperTerminal PE 5.0 Enhancements

  • Guaranteed Y2K-Ready
  • Adds VT220 terminal emulation
  • Includes fixes to avoid video problems introduced by IE 5.0

HyperTerminal PE 4.0 Enhancements

  • Define key macros to issue frequently used series of keystrokes
  • Define the colors used by the terminal screen
  • Set the terminal screen size (rows and columns)
  • Full color support when emulating VT100 terminals
  • Pass-through printing now lets host systems control your printer
  • Ability to exit the program automatically when you log off
  • Includes updated entries for BBSs and systems that you can call
  • Adopts Windows new "flat" toolbar style
  • Uses Windows system sounds (WAV and audio files) instead of beeps
  • Includes bug fixes for improved use with all versions of Windows

Installing and Using HyperTerminal PE

HyperTerminal PE 6.3 is provided as a file called HTPE6.EXE. Simply execute HTPE6.EXE and follow the instructions that display. You can do this by opening the folder that contains HTPE6.EXE and double-clicking on this file, or by selecting Run from the Start menu, browsing to the folder, and selecting this file.

That's it! HyperTerminal PE is now ready to use. You start and use it the same as the HyperTerminal program provided with Windows 95, Windows 98, Windows Me, Windows NT 4.0, and Windows 2000; see your Windows documentation or Help for instructions.

Using HyperTerminal PE as a Telnet Client

Using HyperTerminal PE as a telnet client is simple. You must have TCP/IP installed on your computer first. Once that is done, simply change the "Connect using" setting to "TCP/IP (Winsock)" in HyperTerminal PE's Connect To property sheet, and enter the Host address (either the host name or its numeric IP address and port number). Telnet sites are typically listed in one of the two following forms:

Host name:port number or numeric IP address:port number

If you enter no port number, HyperTerminal PE assumes you mean port number 23, which is the default port for telnet. If the telnet site you want to reach uses a port other than 23, you will need to enter the port number.

Changing the Redial Delay

You may need to change the redial delay if your country restricts how quickly busy numbers may be redialed. (The US has no restrictions.) The default delay is two seconds. To change the delay, you must change a setting in the Windows registry:

1. Run the Registry Editor. If you don't have a shortcut set up for the Registry Editor, type REGEDIT at a DOS prompt.

2. Open the HKEY_LOCAL_MACHINE folder by clicking on the plus sign to the left of the folder, or double-click on the folder.

3. Open the SOFTWARE folder and the Microsoft folder that it contains in a similar fashion.

4. Open the HyperTerminal folder contained in the Microsoft folder.

5. Open the TimeToRedial folder contained in the HyperTerminal folder, and a value will appear in the window on the right.

6. Double-click on the (Default) to bring up the Edit String dialog. Enter the new value and click OK.

7. Exit the Registry Editor.

Using Special Characters

If you find that special characters aren't being sent or displayed correctly by HyperTerminal PE, try changing to a different font. Due to the way Windows maps characters under different fonts, you will see a different character displayed if you are using the Terminal font, as opposed to the other available fonts. There is a more complete discussion of font issues in the FAQs about HyperTerminal PE at http://www.hilgraeve.com/htpefaq.html.
